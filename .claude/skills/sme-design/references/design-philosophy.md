# design-philosophy.md —— 6 個風格庫

**何時載入**：Mode 2（使用者需求模糊、要推方向）時載入，或任何時候您需要幫使用者「挑風格」。這份文件是 sme-design skill 的設計方向手冊，取代 huashu 的 20 風格庫（太重、太西方），精選 6 個台灣中小企業老闆看得懂、拿得出手的風格。

---

## 為什麼只有 6 個

台灣中小企業老闆的審美光譜其實不寬——多數人成長於報紙／雜誌／商業週刊／政府文件，對「專業」的想像集中在少數幾個視覺範式。給 20 個風格只會讓人選擇癱瘓。**6 個風格足以涵蓋 95% 的場景**，剩下 5% 您直接混搭即可。

**選風格不是選「好看」，是選「場合」。** 給銀行經理看的文件、給廟口老闆看的、給合夥人看的、給 ESG 審核人看的，**同一份內容要換三套衣服**。

## 1. 經濟日報社論版

> 當您要老闆「一眼覺得這是正經東西」時用。

**代表調性關鍵詞**：權威、穩重、老派、可信、傳產、安心

**主色區間**：
- 主色：深藍灰 `#1a365d` ~ `#2c3e50`
- 輔色：印刷墨黑 `#1a1a1a`、舊紙米 `#f5f1e8`
- 強調色：暗紅 `#8b2e2e`（限用在關鍵數字／警訊）

**字型建議**：
- 標題：Noto Serif TC Bold 900 / Source Han Serif TC Heavy
- 內文：Noto Sans TC 400-500 / PingFang TC
- 數字：IBM Plex Mono（營收、成長率）

**版面特徵**：
- 頂部細紅線（社論標記）
- 兩欄或三欄文字
- 條列用方塊符號 ▪ 或編號 01 02 03
- 大量橫線分隔區塊，**不用卡片陰影**
- 留白收緊、密度高（像報紙版面）

**代表網站／範本**：
- 經濟日報頭版（udn.com/news/cate/2/6644）
- 工商時報社論版
- Wall Street Journal（英文參考）
- 哈佛商業評論繁中版

**適用場合**：
- 傳產老闆的月報／季報
- 董事會提案
- 銀行融資簡報
- 同業公會投稿
- 給 50 歲以上高階主管看的文件

**避雷**：不要用在給年輕員工看的內部文件，會顯得距離感重、古板。

### 最佳執行路徑

- **字型搭配**：標題 Noto Serif TC 900 + 內文 Noto Sans TC 400-500 + 數字 IBM Plex Mono Medium 500
- **色值公式**：主色 `oklch(0.26 0.06 250)` ≈ `#1a365d`；輔色米底 `oklch(0.93 0.02 85)` ≈ `#f5f1e8`；強調 `oklch(0.40 0.12 25)` ≈ `#8b2e2e`
- **版面**：2-3 欄 grid、橫線分隔（`border-top: 1px solid`）、頂部紅線記號（4px 高）
- **資料呈現**：大量表格靠右對齊數字、編號 01/02/03、Exhibit N 標記圖表
- **陷阱**：不要加陰影／不要圓角 / 不要用彩色背景填滿區塊（紙本思維優先）

## 2. 雜誌特別報導版

> 當您要說「故事」、而不是只報「數字」時用。

**代表調性關鍵詞**：溫潤、人文、敘事、品牌、體驗

**主色區間**：
- 主色：米白 `#faf7f0` ~ `#f5f1e8`
- 文字：深咖啡 `#3d2e1f`、墨灰 `#2a2622`
- 強調色：酒紅 `#722f37`、橄欖綠 `#5c6b3a`、深橘 `#c56b2e`（擇一）

**字型建議**：
- 標題：Playfair Display Italic / Fraunces（英）＋ Noto Serif TC（中）
- 副標：Noto Serif TC Medium
- 內文：Noto Serif TC Regular（放大行距 1.8）

**版面特徵**：
- 大幅照片佔版面 40-60%
- 圖說（caption）用義大利斜體小字
- 首字下沉（drop cap）
- 引言（pull quote）用超大字級 + 單引號
- 頁緣留白充足

**代表網站／範本**：
- 天下雜誌特別報導（cw.com.tw/feature/）
- 遠見雜誌專題
- The New York Times Magazine
- 秋刀魚雜誌（日系繁中）

**適用場合**：
- 品牌故事一頁式
- 客戶訪談／案例研究（case study）
- 年度回顧報告
- 創辦人專訪整理
- 餐飲／文創／生活品牌的對外文件

**避雷**：不要用在給工程師或 B2B 採購看的文件，會顯得「虛」。

### 最佳執行路徑

- **字型搭配**：標題 Playfair Display Italic + 副標 Noto Serif TC Medium + 內文 Noto Serif TC Regular 行距 1.8；caption 用 Italic
- **色值公式**：底色 `oklch(0.96 0.02 85)` ≈ `#faf7f0`；文字 `oklch(0.25 0.04 50)` ≈ `#3d2e1f`；擇一強調 `oklch(0.40 0.15 25)` ≈ `#722f37` 酒紅
- **版面**：單欄或不對稱雙欄、大幅照片佔 40-60%、首字下沉、pull quote 用大字級
- **資料呈現**：圖說小字 Italic、photo caption 放在圖下方左對齊、引言用超大單引號包圍
- **陷阱**：不要塞太多數字／表格（不是 dashboard）／不要用多色 accent（擇一）

## 3. 顧問諮詢所報告版

> 當您要讓客戶覺得「您們像 McKinsey」時用。

**代表調性關鍵詞**：精確、理性、距離、數據、方法論

**主色區間**：
- 底色：冷白 `#ffffff`、淺灰底 `#f7f8fa`
- 主色：海軍藍 `#0f2c4a` 或 企業灰 `#2b2f36`
- 強調色：鮮紅 `#d4001a`（僅用在關鍵對比）
- 輔助：中灰 `#8892a0`、線條灰 `#dfe3e8`

**字型建議**：
- 標題：Instrument Sans Bold / Helvetica Neue Bold（英文強勢；避開 Inter / Inter Tight）
- 中文標題：Noto Sans TC Bold（字重減一級，配合英文視覺平衡）
- 內文：Noto Sans TC 400、行距 1.6
- 數字：IBM Plex Mono（精準對齊，避開 Roboto 系列）

**版面特徵**：
- 明確 grid（8 欄或 12 欄）
- 所有數字靠右對齊
- 圖表極簡——單色為主、沒有 3D、沒有漸變
- 區塊標號 Exhibit 1 / Figure 2.1
- Footer 固定放方法論註記 + 資料來源
- **幾乎不用圓角**（直角是理性的視覺語言）

**代表網站／範本**：
- McKinsey Quarterly（mckinsey.com/quarterly）
- BCG Publications
- 勤業眾信（Deloitte Taiwan）產業報告
- 資策會 MIC 報告模板

**適用場合**：
- B2B 服務業提案
- 市場研究報告（對應 `pmm-market` 的產出）
- 競品深度分析（對應 `pmm-competitive`）
- 給上市櫃公司客戶看的文件
- 投資人 due diligence 資料

**避雷**：不要用在要「打動」讀者的場合，冷白 + 無襯線會顯得缺乏人味。

### 最佳執行路徑

- **字型搭配**：標題 Instrument Sans Bold + 中文 Noto Sans TC Bold（字重減一級）+ 內文 400 行距 1.6 + 數字 IBM Plex Mono
- **色值公式**：底 `oklch(0.98 0 0)` ≈ `#ffffff`；主色 `oklch(0.22 0.08 250)` ≈ `#0f2c4a`；強調紅 `oklch(0.55 0.22 25)` ≈ `#d4001a` 僅用於關鍵對比；線條灰 `#dfe3e8`
- **版面**：12 欄 grid、左右 80px margin、數字靠右、Exhibit N / Figure N.M 編號、footer 固定放方法論
- **資料呈現**：單色圖表（主色變淺）、無 3D、無漸變、表格加 zebra striping（極淺灰）
- **陷阱**：不要襯線（破冷靜感）／不要圓角超過 4px／不要任何裝飾 icon

## 4. 新創 Pitch Deck 版

> 當您要在 30 秒內抓住注意力時用。

**代表調性關鍵詞**：年輕、能量、大膽、斷言、自信

**主色區間**：
- 主色：黑 `#000000` 或 深藍 `#0a0e27`
- 強調色：單一螢光（螢光黃 `#f2f547`、螢光綠 `#00ff94`、熱粉 `#ff2e63`）——**只挑一個**
- 輔色：純白 `#ffffff`

**字型建議**：
- 標題：Archivo Black / Bricolage Grotesque Bold（英文主打；避開 Inter / Inter Tight / Space Grotesk 這些 AI 愛用字）
- 中文：Noto Sans TC Black（字重 900）
- **避開襯線體**——襯線是老派氣質的來源

**版面特徵**：
- 大色塊切版（左半黑、右半螢光）
- 超大數字（180px+）當主視覺
- 一張投影片 = 一句話
- 無圖表時用純文字排版當視覺
- 斜角切版、對齊偶爾故意不對稱（製造張力）

**代表網站／範本**：
- Linear 官網（linear.app）
- Vercel 官網
- AppWorks Demo Day deck 範本
- Y Combinator Pitch Deck 公開樣本

**適用場合**：
- 新創募資簡報
- Demo Day 用 deck
- 招募簡報（給年輕工程師看）
- 品牌 launch 活動投影片

**避雷**：不要用在給傳產老闆或銀行看的文件，會顯得「浮」、不穩重。

### 最佳執行路徑

- **字型搭配**：英文 Archivo Black 或 Bricolage Grotesque Bold、中文 Noto Sans TC 900；避開襯線與 Inter 系列
- **色值公式**：底 `oklch(0 0 0)` 純黑；擇一螢光 `oklch(0.95 0.20 100)` ≈ `#f2f547` 或 `oklch(0.68 0.32 350)` ≈ `#ff2e63`；輔 `#ffffff`
- **版面**：大色塊切版（黑｜螢光）、超大數字 180-240px 當主視覺、一張投影片一句話、偶爾故意不對稱製造張力
- **資料呈現**：數字直接當標題（「$50M」比「年營收 5 千萬」有力）、減少次要資訊、圖表單色 + 螢光 accent
- **陷阱**：不要多個螢光色（只挑一個）／不要襯線／不要細字（全部用 Bold+ 字重）

## 5. 日式極簡商社版

> 當您要賣「高級感」時用。

**代表調性關鍵詞**：細膩、留白、克制、高單價、工藝

**主色區間**：
- 底色：米白 `#fafaf7` 或 象牙白 `#fffef5`
- 主色：墨黑 `#1c1c1c`、水墨灰 `#5a5a5a`
- 強調色：**只用一個** 深藍 `#1a3a5c` 或 朱紅 `#c1272d`（日本傳統色）
- 線條：極細淺灰 `#e8e5de`

**字型建議**：
- 標題：Noto Serif JP Bold + Noto Serif TC Bold（中日混排）
- 內文：Noto Sans TC Light 300（刻意用細字重）
- 數字：Shippori Mincho Regular

**版面特徵**：
- **留白佔版面 50%+**
- 單欄排版（不切多欄）
- 垂直排版重點字（中文豎排）
- 極細分隔線（1px `#e8e5de`）
- 零陰影、零漸變、零裝飾
- 所有字級差異很小（靠字重和位置做層級）

**代表網站／範本**：
- 無印良品官網（muji.com/tw）
- 蔦屋書店（tsite.jp）
- 星野集團官網
- 日本三越伊勢丹 VIP 手冊

**適用場合**：
- 餐飲品牌 menu／品牌書
- 美容沙龍服務手冊
- 高單價服務業（顧問、律師、會計師）
- 茶藝、藝品、精品零售
- 文創品牌型錄

**避雷**：不要用在需要「資訊密度高」的月報或戰卡，會顯得空洞、東西太少。

### 最佳執行路徑

- **字型搭配**：標題 Noto Serif JP Bold + Noto Serif TC Bold 混排、內文 Noto Sans TC Light 300、數字 Shippori Mincho
- **色值公式**：底 `oklch(0.97 0.01 85)` ≈ `#fafaf7`；墨 `oklch(0.20 0 0)` ≈ `#1c1c1c`；擇一強調 `oklch(0.30 0.08 250)` ≈ `#1a3a5c` 或朱紅 `#c1272d`
- **版面**：單欄、留白 50%+、垂直排版重點字、極細 1px 分隔線、零陰影／零漸變／零裝飾
- **資料呈現**：字級差異很小（靠位置和字重做層級）、數字橫排但單位豎排、不用圖表（文字為主）
- **陷阱**：不要加 CTA 按鈕（非商業網站氣質）／不要多色（最多 2 個色相）／不要 icon

## 6. 政府標案版

> 當您要「合規」優於「美觀」時用。

**代表調性關鍵詞**：嚴謹、制式、合規、可稽核、完整

**主色區間**：
- 主色：黑 `#000000`、深藍 `#002060`（Office 預設的深藍）
- 底色：純白 `#ffffff`
- 輔色：政府灰 `#595959`
- **禁用螢光色、禁用漸變**

**字型建議**：
- 標題：標楷體（Biaukai）Bold 或 微軟正黑體 Bold
- 內文：新細明體 / PMingLiU（傳統公文字）或 微軟正黑體 Regular
- **不用 Noto / 西文字型**——評審看到會覺得「不像本土公司」

**版面特徵**：
- A4 縱向，21×29.7cm
- 明確章節編號（壹、貳、參／一、二、三／（一）（二）（三））
- 表格邊框全實線、黑白為主
- 每頁固定 footer：公司抬頭 / 計畫名稱 / 頁碼
- 封面置中、排版對稱
- 章節首頁有大標題 + 分隔頁

**代表網站／範本**：
- 政府電子採購網（web.pcc.gov.tw）公開範本
- 經濟部中小企業處補助申請書範本
- 國發會 SBIR 計畫書範本
- 工研院結案報告模板

**適用場合**：
- 政府補助計畫申請書（SBIR、SIIR、SITI）
- 政府採購投標文件
- 法規遵循報告（ESG、個資、資安）
- 同業公會正式文件
- 法院／律師事務所附件文件

**避雷**：不要用在對外行銷場景，評審以外的人看會覺得「像古董」。

### 最佳執行路徑

- **字型搭配**：標題標楷體 Bold 或微軟正黑體 Bold、內文新細明體 / PMingLiU 或微軟正黑體 Regular；**不用 Noto、不用西文顯眼字**
- **色值公式**：黑 `#000000` + 深藍 `#002060`（Office 預設）+ 政府灰 `#595959`；禁用螢光、禁用漸變
- **版面**：A4 縱向 21×29.7cm、封面置中對稱、章節首頁有分隔頁、明確編號（壹/貳/參 → 一/二/三 → (一)(二)(三)）
- **資料呈現**：表格全實線邊框、黑白為主（關鍵列可用深藍）、每頁固定 footer（公司 / 計畫名 / 頁碼）、目錄頁碼右對齊加虛線
- **陷阱**：不要用 Noto 系列（評審覺得「不像本土公司」）／不要少於 4 級標題層級（審查標案要檢查章節完整）

## 描述情緒而非布局（Mood, Not Layout）

選風格和落地風格時，**描述情緒比規定布局效果更好**。這是給您自己選風格、也給您跟使用者溝通時的共同原則：

| 殺死多樣性的寫法 | 激發品味的寫法 |
|----------------|---------------|
| 「標題置中、兩欄、60% 藍 30% 白」 | 「像禮拜天早上翻經濟日報的從容」 |
| 「字級 48px、行距 1.6、邊距 40px」 | 「像哈佛商業評論中文版那種整潔」 |
| 「紫漸變 + 圓角卡片 + emoji」 | 「像 Linear 產品頁那股斷言的能量」 |

**具體做法**：在 Step 1 文字大綱和 Step 5 variations 的一句話說明裡，**引用具體範例的氣質**（「像 McKinsey 季刊的冷靜」、「像無印良品商品卡的克制」），不要把版型寫死。

為什麼？因為**風格 = 一組內部一致的氣質選擇**，不是一堆獨立參數。您描述氣質，所有參數（字距、邊距、陰影）會自然對齊；您規定參數，氣質反而容易散掉。

## oklch 定義色彩（推薦做法）

當您要從選定風格的「主色區間」產生深淺變體、或調整飽和度時，**用 oklch 不要用 hsl/hex**：

```css
:root {
  /* 主色用 oklch 定義，衍生色自動和諧 */
  --brand-primary: oklch(0.35 0.09 250);        /* 深藍灰 */
  --brand-primary-light: oklch(0.55 0.09 250);  /* 同色相變亮 */
  --brand-primary-dark: oklch(0.22 0.09 250);   /* 同色相變暗 */

  /* 強調色 */
  --brand-accent: oklch(0.55 0.18 25);          /* 暗紅 */
}
```

**為什麼 oklch 比 hsl 好**：oklch 調整亮度時色相不漂移（hsl 會），產出的明暗變體看起來是「同一個顏色的不同深度」，不是「不同顏色」。這是現代設計系統的標配。

**但 hex 仍是交付通貨**：跟使用者溝通和寫進 brand-spec.md 時，仍用 hex（老闆 / 印刷廠 / PPT 都要 hex）。oklch 是您內部計算的工具。

## 跳出 6 風格時的情緒參考

6 個風格已涵蓋 95% 場景，剩下 5% 您可能需要混搭或引用其他氣質。以下是幾個值得收藏的氣質原型，**不是額外風格庫**，是當您要描述「我要那種感覺」時可以引用的語彙：

- **出版物氣質**（editorial） — 像 The New Yorker / 天下雜誌專題 / Bloomberg Businessweek 那種「一篇文章長在紙上」的感覺
- **瑞士網格派**（Swiss grid） — 像 Müller-Brockmann 海報那種數學化的冷靜，標題、內文、數字都在 8pt baseline 上
- **東方留白派**（Kenya Hara / 無印） — 留白佔 70%+、只用一個色相、所有元素克制到近乎看不見
- **展覽館氣質**（Apple Gallery） — 淺色底 + 悬浮卡片 + 软阴影 + 1 個 accent 色相，像美術館的陳列牆
- **新聞印刷氣質** — 襯線 + 双线分隔 + eyebrow label + 細紅線標記，像報紙頭版
- **報告書氣質**（corporate reporting） — 大量表格 + 腳註 + Exhibit 編號 + Footer 方法論，像 McKinsey/BCG 年度刊物

**用法**：跟使用者說「這次要做的是『出版物氣質』，像天下雜誌那種，不是『新創 Pitch』那種」—— 使用者一聽就懂，比規定「字級」清楚 10 倍。

## 快速參考

1. **選風格前先問一句**：「這份文件誰看？在什麼場合看？」—— 答案直接對應 6 個風格之一
2. **不要混兩個風格**：想強調「既專業又溫潤」會兩個都不像，選一個做到位
3. **品牌色優先於風格色**：有 brand-spec.md 時，把該風格的主色位置換成品牌色
4. **預設推 3 個方向**：對使用者展示時給 3 個選項（例：`社論版` vs `顧問版` vs `雜誌版`），讓選擇產生 ownership
5. **給資料不足時**：預設用「經濟日報社論版」—— 台灣中小企業最安全的起手式
6. **描述情緒而非布局**：引用具體範例的氣質（「像經濟日報」）比規定參數有效
7. **色彩用 oklch 衍生深淺**，但對外交付仍用 hex
