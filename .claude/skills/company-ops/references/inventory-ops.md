# 庫存管理專業指南

## 觸發情境

「A200 還有多少」「進貨 50 個」「哪些快沒了」「盤點」
「呆滯品」「庫存週轉」「該補貨了嗎」「哪些賣最好」

## 一、查詢庫存

- 指定品項 → `check_stock(sku_or_name)`
- 所有警報 → `low_stock_alerts()`
- 模糊搜尋 → `check_stock(關鍵字)`

### 新品項建檔

新品項可透過 `update_stock` 一次建立 SKU + 設定品項資訊 + 初始庫存：

```
update_stock(
  sku='SKU-A001',
  quantity_change=50,
  reason='初始建檔',
  name='香氛蠟燭',
  sell_price=500,
  unit_cost=200,
  min_stock=20,
  unit='個',
  category='香氛'
)
```

SKU 已存在時只更新數量，忽略其他參數。

## 二、ABC 分析（庫存分級管理）

### 分類原則

| 等級 | 佔品項數 | 佔銷售金額 | 管理強度 |
|------|---------|-----------|---------|
| A 級 | 約 10-20% | 約 70-80% | 高：每週檢查，精確控制安全庫存 |
| B 級 | 約 20-30% | 約 15-20% | 中：每兩週檢查，標準安全庫存 |
| C 級 | 約 50-70% | 約 5-10% | 低：每月檢查，簡化管理 |

### 各級管理策略

**A 級品項：**
- 安全庫存：精確計算（見下方公式）
- 盤點頻率：每週或每兩週
- 補貨策略：頻繁小量，避免積壓資金
- 監控：庫存異動即時通知

**B 級品項：**
- 安全庫存：用簡化公式
- 盤點頻率：每月
- 補貨策略：定期定量

**C 級品項：**
- 安全庫存：概略數字即可
- 盤點頻率：每季
- 補貨策略：大量少次，降低訂購成本

## 三、安全庫存與補貨點

### 安全庫存計算（簡化版）

```
安全庫存 = 日均銷量 × 供應商交貨天數 × 波動係數
```

| 波動係數 | 適用情況 |
|---------|---------|
| 1.0 | 銷量穩定，供應商準時 |
| 1.3 | 銷量略有波動 |
| 1.5 | 銷量波動大或供應商偶爾延遲 |
| 2.0 | 銷量極不穩定或供應商常延遲 |

**範例：** 日均賣 5 個，供應商要 7 天交貨，銷量略有波動
→ 安全庫存 = 5 × 7 × 1.3 = 46 個

### 補貨點計算

```
補貨點 = 安全庫存 + (日均銷量 × 交貨天數)
```

**範例：** 安全庫存 46，日均銷 5，交貨 7 天
→ 補貨點 = 46 + 35 = 81 個
→ 庫存降到 81 就該下單

### 建議補貨量

- A 級品項：補 7-14 天的量（少量多次）
- B 級品項：補 14-30 天的量
- C 級品項：補 30-60 天的量

### 自動補貨提醒

當 `low_stock_alerts()` 回傳警報時：
- A 級且 ≤ 補貨點 → 立即通知老闆，含建議補貨量
- B 級 → 加入本週補貨清單
- C 級 → 加入本月補貨清單
- 庫存 = 0 → 緊急通知：「{品名} 已斷貨！」

## 四、進貨流程

1. `check_stock` 確認目前庫存
2. 計算進貨金額（數量 × unit_cost）
3. 金額超過審核門檻？
   - 是 → `create_approval` + LINE 通知主管
   - 否 → 直接執行
4. `update_stock(sku, +數量, 原因)`
   > ⚠️ `update_stock` 的回傳值會包含 `👉 下一步` 提示建議的 `record_transaction` 參數（含計算好的金額）。**一定要記帳**，否則進貨金額不會出現在帳務中。
5. 同時記帳（跨模組串接 → accounting-ops）：
   - 問使用者：「已付款還是賒帳？」
   - 已付款 → `record_transaction(type='expense', amount=進貨金額, category='inventory_purchase', description='進貨 {品名} {數量} 個 @ NT${單價}', payment_status='paid')`
   - 賒帳 → `record_transaction(type='expense', amount=進貨金額, category='inventory_purchase', description='進貨 {品名} {數量} 個 @ NT${單價}', payment_status='pending', due_date=帳期到期日)`
6. 如果有進貨單/發票照片 → `add_attachment(target_type='inventory', target_id=品項ID, file_path='data/media/inventory/進貨單.jpg', description='進貨單')`
7. 回報：「已入庫 {品名} {數量} 個，目前庫存 {新數量}。已記錄進貨支出 NT${金額}。」

### 進貨驗收提醒

提醒使用者確認：
- [ ] 數量與訂單相符
- [ ] 外觀/品質無異常
- [ ] 有效期限充足（如適用）
- [ ] 進貨單/發票已取得

## 五、出貨 / 損耗

### 訂單出貨（優先走此流程）

如果出貨對應一張訂單，請走 **order-ops** 的出貨流程：
`qc_order` → `fulfill_order` → 自動扣庫存 + 自動建應收帳款。
**不要再手動 `update_stock`**，否則會重複扣庫存。

### 非訂單出貨 / 損耗

沒有對應訂單的手動扣庫存（損耗、贈品、樣品等）：

1. `check_stock` 確認夠不夠
2. 庫存不足 → 回報：「目前只有 {X} 個，不夠出 {Y} 個」
3. 庫存足夠 → `update_stock(sku, -數量, 原因)`
4. 扣完低於安全庫存 → 警報通知
5. 扣完低於補貨點 → 建議補貨

## 六、盤點作業

### 循環盤點計畫

| 品項等級 | 盤點頻率 | 方式 |
|---------|---------|------|
| A 級 | 每週或每兩週 | 全數清點 |
| B 級 | 每月 | 全數清點 |
| C 級 | 每季 | 抽點 30-50% |
| 全面盤點 | 每半年 | 所有品項 |

### 盤點流程（員工 LINE 回報）

員工傳「A200 實際 8 個」：
1. 解析品項和數量
2. `check_stock('A200')` 取得帳面數量
3. 比對差異
4. 差異 = 0 → 「A200 帳實相符 ✅」
5. 差異 ≠ 0 → 「帳面 {X}，實際 {Y}，差異 {Z}。要調整嗎？」
6. 確認 → `update_stock(sku, 差異數, '盤點調整')`

### 盤點差異處理

| 差異率 | 處理 |
|--------|------|
| ≤ 2% | 正常，直接調整 |
| 2-5% | 調整，但標記需調查 |
| 5-10% | 通知主管，先複盤再調整 |
| > 10% | 緊急通知，暫停出貨，徹底清查 |

### 庫存準確率

```
庫存準確率 = 帳實相符品項數 / 盤點品項總數 × 100%
```

| 準確率 | 評價 |
|--------|------|
| ≥ 98% | 優秀 |
| 95-97% | 良好 |
| 90-94% | 需改善 |
| < 90% | 嚴重，可能有系統性問題 |

## 七、呆滯品管理

### 識別標準

| 類型 | 定義 |
|------|------|
| 慢動品 | 90 天內無銷售 |
| 呆滯品 | 180 天內無銷售 |
| 過期品 | 已超過有效期限 |

### 處理決策樹

```
還能賣嗎？
├→ 能：
│   ├→ 折扣促銷（7 折起）
│   ├→ 搭配暢銷品組合
│   └→ 轉換通路（線上/批發）
└→ 不能：
    ├→ 報廢，記錄損失
    └→ record_transaction(type='expense', category='inventory_loss')
```

### 呆滯品資金佔用

| 佔用比例 | 評價 | 建議 |
|---------|------|------|
| < 5% | 正常 | 持續監控 |
| 5-15% | 偏高 | 啟動促銷清貨 |
| > 15% | 嚴重 | 立即處理 |

## 八、季節性需求（台灣）

| 月份 | 特徵 | 備貨重點 |
|------|------|---------|
| 1-2月 | 農曆年前旺→年後淡 | 年前增備，年後控制 |
| 3-4月 | 清明、復甦期 | 正常 |
| 5-6月 | 端午、畢業季 | 相關商品提前備 |
| 7-8月 | 暑假、鬼月（部分行業淡） | 視行業 |
| 9-10月 | 中秋、開學 | 提前 1 個月備 |
| 11-12月 | 雙11、聖誕、跨年 | 旺季備貨，注意物流延遲 |

旺季前 4-6 週：A 級品項安全庫存 +30-50%
旺季結束後：恢復正常，清理剩餘庫存

## 九、供應商交貨追蹤

每次進貨記錄：下單日、預計到貨日、實際到貨日、延遲天數

| 準時率 | 評價 | 波動係數建議 |
|--------|------|------------|
| ≥ 95% | 可靠 | 1.0 |
| 80-94% | 尚可 | 1.3 |
| 60-79% | 不穩定 | 1.5-2.0 |
| < 60% | 不可靠 | 建議換供應商 |

## 十、庫存週轉率

```
月庫存週轉率 = 月銷售成本 / 平均庫存金額
年庫存週轉率 = 月週轉率 × 12
```

| 年週轉率 | 一般零售業 |
|---------|----------|
| > 12 次 | 優秀 |
| 6-12 次 | 良好 |
| 3-6 次 | 普通，可能有呆滯品 |
| < 3 次 | 偏低，需檢討 |

## 十一、常見錯誤

| 錯誤 | 預防 |
|------|------|
| 進出貨不即時記錄 | 每次異動立刻更新 |
| 安全庫存設太高 | 用公式算，每季檢討 |
| 安全庫存設太低 | 考慮供應商交期波動 |
| 新品一次進太多 | 首批只進 2 週預估量 |
| 不做盤點 | 至少每月盤 A 級品項 |
| 憑感覺補貨 | 用資料決定 |

## Do's and Don'ts

### Do
- 每次庫存變動即時更新，不要延後
- 進貨後一定跟著 `record_transaction`（`update_stock` 回傳值有提示）
- 大量調整（盤點差異 > 5%）需主管核准
- 新品首批只進 2 週預估量
- 安全庫存用公式計算，每季檢討

### Don't
- 不要在 `fulfill_order` 之後再手動 `update_stock`（自動扣庫存，重複會錯）
- 不要允許負庫存（系統直接拒絕）
- 不要讓 basic 權限的人新增品項（manager 以上）
- 不要憑感覺補貨（用資料和補貨點公式）
- 不要跳過進貨記帳步驟

## 快速參考

### 進貨
1. `check_stock(sku_or_name='SKU-A001')` — 確認目前庫存
2. `update_stock(sku='SKU-A001', quantity_change=50, reason='供應商到貨')`
3. 依回傳的「👉 下一步」→ `record_transaction(type='expense', amount=進貨金額, category='inventory_purchase')`

### 盤點調整
1. `check_stock(sku_or_name='A200')` — 取帳面數量
2. 比對實際數量，計算差異
3. 差異 ≤ 2% → `update_stock(sku='A200', quantity_change=差異數, reason='盤點調整')`
4. 差異 > 5% → 通知主管，先複盤再調整

## 中斷恢復

如果 context 被壓縮：
1. `get_context_summary(scope='full')` — 查看「庫存警報」區塊
2. `low_stock_alerts()` — 完整列出所有低於安全庫存的品項
3. 對每個警報品項 `check_stock(sku)` 取得詳情
4. 根據 ABC 分級和補貨點判斷：立即通知/本週補貨/本月補貨
