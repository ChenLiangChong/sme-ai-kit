# 報表生成專業指南

## 觸發情境

「這個月的報表」「週報」「業績分析」「匯出」「今天做了什麼」

---

## 一、報表類型

### 日報

觸發：使用者主動要求；或 CLAUDE.md 啟動流程時由 agent 判斷該不該推（系統目前未自動排程或定時推送）

```
{公司名稱} 日報 — {日期}

📝 任務
  今日完成：X 項
  今日新增：X 項
  明日到期：X 項

💬 LINE 互動
  收到：X 則
  回覆：X 則

💹 今日收支
  收入：NT$ X
  支出：NT$ X

⚠️ 庫存警報：X 項（如有列出）
⏳ 待審核：X 項（如有列出）
```

### 週報

觸發：使用者說「週報」（系統不會每週五自動提醒；要定時得靠未來的排程器）

```
{公司名稱} 週報 — 第 XX 週（MM/DD - MM/DD）

📝 任務
  完成：X 項（完成率 XX%）
  新增：X 項
  逾期：X 項
  下週到期：X 項

👤 客戶
  新增客戶：X 位
  LINE 互動：X 則
  客訴：X 件（已處理 X 件）

💹 收支
  收入：NT$ XXX,XXX（上週 +X%/-X%）
  支出：NT$ XXX,XXX（上週 +X%/-X%）
  淨額：NT$ ±XX,XXX

📦 庫存
  補貨：X 次
  警報：X 項
  盤點準確率：XX%（如本週有盤）

🔑 重點事項
  - {本週最重要的 1-3 件事}

📋 下週重點
  - {下週最需要關注的 1-3 件事}
```

### 月報

觸發：使用者說「月報」（系統不會每月 1 日自動提醒做月結；要定時得靠未來的排程器）

```
{公司名稱} 月報 — {YYYY}年{MM}月

💹 財務摘要
  收入：NT$ XXX,XXX（上月 +X%）
  支出：NT$ XXX,XXX（上月 +X%）
  淨利：NT$ ±XX,XXX
  毛利率：XX%
  淨利率：XX%

  支出結構
  - 進貨成本 XX%
  - 薪資 XX%
  - 租金 XX%
  - 其他 XX%

📝 任務統計
  總任務：X 項
  完成率：XX%
  準時率：XX%
  平均完成天數：X 天

👤 客戶統計
  總客戶：X 位（+X 新客）
  活躍客戶（60天內有消費）：X 位
  流失風險（60天以上沒消費）：X 位
  客訴：X 件，解決率 XX%

📦 庫存統計
  庫存總金額：約 NT$ XXX,XXX
  庫存週轉率：X.X 次
  呆滯品佔比：XX%
  庫存準確率：XX%

📋 企業規則變更
  {knowledge_changelog(days=30) 摘要}
  新增：X 條 | 更新：X 條

🔍 知識健檢
  {lint_knowledge() 摘要 — 矛盾 X 組、過期 X 條、覆蓋缺口 X 個}

🔑 本月亮點
  - {最好的 1-3 件事}

⚠️ 需要關注
  - {需要改善的 1-3 件事}
```

## 三、數據解讀原則

### 不只報數字，要給洞察

| 差的報表 | 好的報表 |
|---------|---------|
| 「本月收入 NT$500,000」 | 「本月收入 NT$500,000，比上月成長 12%，主要來自 A 產品線的經銷商訂單增加」 |
| 「庫存週轉率 4.2」 | 「庫存週轉率 4.2（去年同期 3.8），改善主因是 Q1 清理呆滯品的成效」 |
| 「客訴 5 件」 | 「客訴 5 件（上月 3 件），其中 3 件是物流延遲，建議跟物流商檢討」 |

### 比較維度

每個指標盡量附帶至少一個比較：
- 上月（MoM）
- 去年同月（YoY，如有資料）
- 目標值（如有設定）
- 業界平均（如知道）

### 異常標記

| 變動幅度 | 標記 |
|---------|------|
| ±5% 以內 | 正常，不特別標記 |
| ±5-15% | 📈 / 📉 標記趨勢 |
| ±15-30% | ⚠️ 標記為異常 |
| ±30% 以上 | 🔴 標記為重大異常，附帶解釋 |

## 四、輸出方式

### LINE 精簡版

用 `reply` 發送 3-5 個核心數字。
前綴加 `[日報]` / `[週報]` / `[月報]`。
不超過 500 字。

### 對話詳細版

在 Claude 對話中以 markdown 呈現完整報表。

### 檔案匯出

老闆要求「匯出」「完整版」「Excel」「PDF」時，使用 office skill 產出檔案：

| 格式 | 使用的 Skill | 適用場景 |
|------|-------------|---------|
| .xlsx | xlsx skill | 月報、帳務明細、庫存報表（含公式和格式） |
| .pdf | pdf skill | 報價單、發票、正式報告 |
| .docx | docx skill | 合約、正式文件、客戶提案 |

操作流程：
1. 用 MCP tool 撈取數據（`monthly_summary`、`list_transactions` 等）
2. 整理成結構化資料
3. 載入對應的 office skill → 按 SKILL.md 流程產出檔案
4. 檔案產出到 `data/media/exports/` 目錄，命名格式：`{日期}_{報表名}.{格式}`
   例如：`data/media/exports/2026-04_月報.xlsx`
5. 回報：「報表已產出：{檔案路徑}」

Python 環境：`.venv/bin/python3`（已安裝 openpyxl、pypdf、python-pptx 等依賴）

## 五、推送時機（非系統排程）

**系統目前沒有報表排程器、也不會定時推 LINE。** 報表一律是「使用者要求時產出」，或在 **CLAUDE.md 啟動流程**被觸發時由 agent 自行判斷該不該主動提一句、屬 agent 後續動作而非系統定時任務：
- 日報：使用者要求時產出；agent 在啟動流程若判斷今天還沒給過、可主動提
- 週報：使用者說「週報」時產出；agent 可在週末前後判斷要不要提醒老闆看（非系統每週五推）
- 月報：使用者說「月報」時產出；agent 可在月初判斷要不要提醒做月結（非系統每月 1 日推）

要做到「每天早上自動推 / 每週五自動提醒 / 每月 1 日自動提醒」這類定時推送、需等未來補上排程器；現況靠 agent 在被喚醒（啟動 / 收到 LINE 訊息）的當下判斷，沒有背景定時觸發。

## Do's and Don'ts

### Do
- 數據全來自 DB，找不到的區塊標注「無資料」而非跳過
- 每個指標附帶至少一個比較（上月 MoM、目標值）
- 不只報數字，要給洞察和可能原因
- 異常變動 > ±15% 標記 ⚠️，> ±30% 標記 🔴 並附解釋

### Don't
- 不要編造數字或推測沒有資料支撐的趨勢
- 不要跳過數據為零或找不到資料的區塊
- 不要在資料不足 2 個月的情況下做趨勢分析
- 不要做純讀取以外的操作（report-gen 不寫入）

## 快速參考

### 日報
1. `get_context_summary(scope='full')` — 系統全貌
2. `list_transactions(start_date='<今天 YYYY-MM-DD>', end_date='<今天 YYYY-MM-DD>')` — 今日收支（**工具不解析「今天」這類自然語言、日期要傳實際 `YYYY-MM-DD` 字串**，agent 先換算當天日期再傳）
3. 整理成日報格式 → LINE 推送或回覆老闆

### 週報
1. `list_tasks(status='done')` + `list_tasks(status='pending')` — 任務統計
2. `list_transactions(start_date='<本週一 YYYY-MM-DD>', end_date='<今天 YYYY-MM-DD>')` — 週收支（同上、agent 先把「本週一/今天」換算成實際 `YYYY-MM-DD` 再傳）
3. `low_stock_alerts()` — 庫存警報
4. 整理含「🔑 重點事項」和「📋 下週重點」

### 月報 + 匯出
1. `monthly_summary(year_month='2026-03')` — 月度收支彙總
2. 各維度統計（任務、客戶、庫存）
3. 需要匯出 → 載入 xlsx skill → 產出到 `data/media/exports/`

## 六、注意事項

- 純讀取，不做寫入
- 數據全來自 DB，不要編造
- 找不到資料的區塊標注「無資料」而非跳過
- 跟 ops-dashboard 的差異：dashboard 是即時快照，report-gen 是期間彙總
- 報表中的趨勢分析至少需要 2 個月以上的資料才有意義
- **floor 可見度**：財務區塊（`monthly_summary` / `list_transactions` / `check_overdue`）只在財務可見層（`financial_visibility=all` 或全權限層）跑得出來——floored 非財務層這些工具被移除、產不出收支 / 財務摘要屬正常（呈現「本層不可見」、非當缺資料）。（註：目前**只有 `query_knowledge` 保證機密過濾**；`knowledge_changelog` / `lint_knowledge` 不做 floor 過濾，月報的「規則變更 / 知識健檢」區塊一律在全權限層產出。庫存警報 `low_stock_alerts` 在非全權限層只回精簡視圖、非完整清單[#166]。）見 CLAUDE.md〈部門安全層（floor）與兩道牆〉
