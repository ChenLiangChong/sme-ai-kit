# 品牌語氣專業指南

## 觸發情境

「幫我寫信給客戶」「草擬公告」「行銷文案」「回覆客訴」「品牌設定」

---

## 一、定位

品牌語氣是**裝飾器** — 其他模組產出對外文案後，套用品牌語氣再發出。
不獨立運作，而是被 crm-ops、line-comms 等模組呼叫。

---

## 二、品牌語氣設定（初次導入）

### 方式一：範例文案分析（推薦）

請老闆提供 3-5 篇過去寫得好的信件/文案/社群貼文。

分析維度：
1. **詞彙偏好** — 常用哪些詞？避免哪些詞？
2. **句子結構** — 長句還是短句？用不用問句？
3. **稱呼方式** — 您/你/親愛的/寶貝？
4. **情感基調** — 專業/溫暖/幽默/簡約？
5. **特殊慣例** — 固定開場白？結尾用語？

分析完後存入 DB：
```
store_fact(
  category='brand',
  title='品牌語氣分析',
  content='根據分析，品牌語氣為...',
  source_type='observed',
  set_by='系統分析'
)
```

### 方式二：互動問答（快速版）

用以下 5 個問題快速定義：

1. **用 3 個詞描述你的品牌個性**
   例：「專業、溫暖、值得信賴」

2. **你欣賞誰的溝通風格？**
   例：「無印良品的簡約」「誠品的文藝」

3. **你常用什麼詞或口頭禪？**
   例：「超值」「精心挑選」「為你打造」

4. **哪些詞絕對不要用？**
   例：「親」「寶貝」「趕快搶購」

5. **句子風格偏好？**
   例：「短句，直接，不繞彎」

每個回答都用 `store_fact(category='brand')` 存入。

### 方式三：兩者結合（最完整）

先分析範例 → 再問答補充 → 產出完整品牌語氣指南。

## 三、語氣場景矩陣

不同對象 × 不同場景，用不同的語氣：

| 場景 | 對客戶 | 對供應商 | 對員工 |
|------|--------|---------|--------|
| 稱呼 | 您 | 貴公司 | 你 |
| 基調 | 親切溫暖 | 專業明確 | 直接簡潔 |
| 用語 | 白話、生活化 | 商業術語 OK | 口語 OK |
| 範例 | 「感謝您的支持」 | 「請貴公司確認」 | 「這個幫我處理一下」 |

### 特殊場景語氣

| 場景 | 語氣調整 |
|------|---------|
| 催款 | 堅定但有禮 |
| 客訴回覆 | 同理心優先，不辯解 |
| 壞消息 | 直說但提供替代方案 |
| 好消息 | 熱情但不浮誇 |
| 促銷 | 有吸引力但不誇大 |

## 四、文案框架

### AIDA 框架（行銷觸及用）

| 步驟 | 做什麼 | 範例 |
|------|--------|------|
| **A**ttention 注意力 | 開頭一句話抓住眼球 | 「你知道嗎？90% 的人都忽略了這件事」 |
| **I**nterest 興趣 | 連結到他的痛點 | 「每天花 2 小時整理資料，其實可以只要 10 分鐘」 |
| **D**esire 渴望 | 展示解決方案的好處 | 「我們的客戶平均省下 60% 的時間」 |
| **A**ction 行動 | 明確的下一步 | 「回覆『了解』我們立刻為你安排」 |

### PAS 框架（客訴/問題解決用）

| 步驟 | 做什麼 |
|------|--------|
| **P**roblem 問題 | 描述客戶遇到的困擾 |
| **A**gitate 強化 | 說明如果不解決會怎樣 |
| **S**olve 解決 | 提供你的解決方案 |

### BAB 框架（改變說服用）

| 步驟 | 做什麼 |
|------|--------|
| **B**efore 之前 | 描述現在的痛苦狀態 |
| **A**fter 之後 | 描述改善後的美好狀態 |
| **B**ridge 橋樑 | 你的產品/服務就是那座橋 |

## 五、寫作品質檢查清單

每篇對外文案完成後，逐項確認：

### 基本檢查
- [ ] 符合品牌語氣設定（從 DB 讀取）
- [ ] 沒有用到禁用詞
- [ ] 稱呼方式正確（對象是誰？）
- [ ] 行動呼籲（CTA）明確

### 台灣中文用語
- [ ] 用「資料」不用「數據」
- [ ] 用「軟體」不用「軟件」
- [ ] 用「透過」不用「通過」
- [ ] 用「取得」不用「獲取」
- [ ] 沒有中國式用語（「親」「寶貝」「小哥哥」）

### LINE 訊息特別注意
- [ ] 一則訊息不超過 200 字
- [ ] 重點放前面（手機螢幕小）
- [ ] 適當換行，不要一大坨
- [ ] emoji 用但不過量（1-2 個就好）

## 六、常用範本庫

### 範本結構

每個範本存在 business_rules 表，category='brand'：

| 範本名 | 場景 | 存入方式 |
|--------|------|---------|
| 感謝信 | 購買後致謝 | `store_fact(category='brand', title='範本-感謝信')` |
| 催款信 | 應收帳款提醒 | `store_fact(category='brand', title='範本-催款信')` |
| 節日問候 | 過年/中秋/端午 | `store_fact(category='brand', title='範本-節日問候')` |
| 新品通知 | 新產品上架 | `store_fact(category='brand', title='範本-新品通知')` |
| 客訴回覆 | 處理客訴後 | `store_fact(category='brand', title='範本-客訴回覆')` |
| 歡迎新客 | 首次購買 | `store_fact(category='brand', title='範本-歡迎新客')` |

### 範本使用流程

1. 判斷場景 → 從 DB 找對應範本
2. 替換變數（{客戶名}、{產品名}、{金額}）
3. 套用當前語氣場景（對客戶/對供應商）
4. 對外發送 → 送審

## 七、跨模組協作

| 模組呼叫 brand-voice 的時機 | 做什麼 |
|---------------------------|--------|
| crm-ops 要發行銷訊息 | 套用品牌語氣 + 選框架（AIDA/PAS） |
| line-comms 回覆客戶 | 套用對客戶語氣 |
| report-gen 產出對外報告 | 套用專業語氣 |
| onboarding 歡迎新員工 | 套用對員工語氣 |

---

## 八、注意事項

- 對外文案一律送審
- 不要擅自改變老闆設定的品牌風格
- DB 沒有品牌規則 → 用中性專業語氣，建議設定
- 範本只是起點，每次都要根據具體情境微調
- 不要過度使用 emoji 和驚嘆號
