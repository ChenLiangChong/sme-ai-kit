# PMM Gate 5：競品情報

Battlecard（競品戰卡）、競品態勢分析、Win/Loss（成敗分析）追蹤與競品動態監控。

> **對應 agent**：`.claude/agents/competitive-strategist.md`（thin proxy）
> 本檔是方法論的**單一來源**；agent 提供結構化 I/O schema 強制 + 紀律框架。修改本檔內容時，agent 自動受惠。
> Battlecard 視覺設計（HTML / 一頁式 PPT）由 `sme-design` skill 處理。

---

> **工具可用性**：本模組會嘗試呼叫 `query_knowledge`（屬 business-db MCP）載入既有品牌／競品／市場脈絡。若該工具不可用（例如第一堂課只裝了 Claude Desktop + social-media skill），請直接向使用者詢問所需資訊，不要因工具缺失而停止。
>
> **最小問題清單（沒 MCP 時先問這些）：**
> - 您的公司／品牌是什麼？主要產品或服務？
> - 目標客戶是誰？（規模、行業、痛點）
> - 目前的行銷／定位狀況？有什麼既有資料可以參考？
> - 前三大競品是誰？

## 競品情報流程

建立競品知識庫：

1. 盤點 Tier 1（直接競品）、Tier 2（相鄰方案）、Tier 3（現狀做法）
2. 註冊競品產品（親自動手評估）
3. 監控競品官網、定價、訊息框架
4. 分析業務通話錄音，擷取競品被提及的片段
5. 閱讀 G2 / Capterra 評論（優缺點）
6. 追蹤競品徵才公告（Roadmap 訊號）
7. 每月更新 Battlecard
8. **驗證標準：** 業務團隊在 80% 以上的競爭案件中使用 Battlecard

### 競品分層結構

| Tier | 定義 | 範例 |
|------|------|------|
| 1 | 直接競品、同品類 | [競品 A、B] |
| 2 | 相鄰方案、使用情境重疊 | [替代方案 C、D] |
| 3 | 現狀（客戶目前的做法） | Excel、手動作業、自建系統 |

3. [不足之處 + 證據]

我方優勢 (Our Advantages)：
1. [我方優勢 + 證據 / 數據]
2. [我方優勢 + 證據 / 數據]
3. [我方優勢 + 證據 / 數據]

## Win/Loss 分析

### 每月追蹤

| 指標 | 本月 | 上月 | 趨勢 |
|------|------|------|------|
| 整體勝率 | X% | X% | ↑/↓ |
| 對 [競品 A] 勝率 | X% | X% | ↑/↓ |
| 對 [競品 B] 勝率 | X% | X% | ↑/↓ |
| 對現狀勝率 | X% | X% | ↑/↓ |
| 首要勝因 | [原因] | [原因] | — |
| 首要敗因 | [原因] | [原因] | — |

### Win/Loss 訪談題目

1. 「您當時評估了哪些替代方案？」
2. 「您做決定時最重要的因素是什麼？」
3. 「差點讓您選擇 [競品] 的原因是什麼？」
4. 「什麼情況會改變您的決定？」
5. 「各方案的定價比較起來如何？」
6. 「決策過程中還有誰參與？」
7. 「在您開始評估之前，原本的做法是什麼？」
8. 「您最初是怎麼知道各個選項的？」

### 分析框架

追蹤並歸類：
- **勝因**：產品契合、易用性、價格、支援、品牌、整合
- **敗因**：功能缺失、價格、關係、既有領導者、時機
- **行動項目**：分派給產品團隊、業務團隊、行銷團隊

## 競品動態回應 Playbook

### 競品推出新功能時

1. **評估影響**（當天）：是否影響我方差異化？
2. **內部簡報**（24 小時內）：向業務團隊說明新功能的意義
3. **話術準備**（48 小時內）：客戶詢問時如何回應
4. **Battlecard 更新**（1 週內）：納入競品素材
5. **策略回應**（2 週內）：決定是否需要調整產品或定位

### 競品改變定價時

1. **記錄變動**：新舊價格、方案組合變化
2. **分析影響**：是否影響我方價格定位？
3. **更新比較素材**：修訂所有定價比較文件
4. **評估回應**：我方是否需要調整？（通常不用，避免反射性定價）
5. **業務簡報**：提供價格異議的話術

### 競品取得媒體曝光時

1. 不要恐慌——媒體報導 ≠ 市場佔有率
2. 分析報導角度：他們主打什麼敘事？
3. 準備反敘事，以備記者來訪
4. 若時機合適，加速自身 PR 進度
5. 當作動力來源，不要變成反射性變更的觸發點

## Proactive Triggers（主動觸發）

- **偵測到競品重新定位** → 檢視您的定位是否仍成立。
- **新競品進入市場** → 加入競品態勢圖、評估分層。
- **對特定競品的勝率下滑** → 深入分析、更新 Battlecard。
- **業務團隊詢問「如何跟 X 競爭？」** → 該產 Battlecard 了。

## Do's and Don'ts

### Do
- Battlecard 每月更新 — 過期的情報比沒有情報更危險
- 親自試用競品產品 — 註冊、操作、體驗完整流程，不要只看官網
- 三層分類競品：直接競品（Tier 1）、相鄰方案（Tier 2）、現狀（Tier 3）
- 追蹤競品的徵才訊息 — 這是預測對手 roadmap 的重要信號

### Don't
- 不要只靠二手資訊了解競品 — 官網文案 ≠ 真實產品體驗，一定要親自試用
- 不要忽略 Tier 3「現狀」競品 — Excel、手動流程、不改變才是最大的敵人
- 不要讓 Battlecard 超過一頁 — 業務不會讀長文件，一頁精華才有用
- 不要因為競品得到媒體報導就恐慌 — 報導 ≠ 市占率，保持戰略節奏

## 快速參考

### 建立競品情報庫
1. 列出 Tier 1/2/3 競品清單
2. 每個 Tier 1 競品：註冊試用、記錄定價、截圖關鍵功能
3. 收集 G2/Capterra 評論，歸納優缺點
4. 建立 Battlecard（每個競品一頁），包含定位、定價、優劣勢、話術
5. 設定月度更新排程

### Win/Loss 分析
1. 每月統計 vs 各競品的勝率
2. 訪談贏單/丟單客戶（至少問 8 題標準問題）
3. 歸類勝因/敗因（產品、價格、關係、時機）
4. 產出行動項目給產品、業務、行銷團隊

### 競品動態回應
1. 競品發新功能 → 當天評估影響 → 24h 內部簡報 → 1 週更新 Battlecard
2. 競品改價格 → 記錄新舊價差 → 評估是否影響我方定位 → 更新比較素材
3. 競品上新聞 → 分析報導角度 → 準備反敘事 → 不要反射性跟進
