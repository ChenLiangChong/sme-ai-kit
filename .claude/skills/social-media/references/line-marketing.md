# LINE 行銷策略參考

LINE 官方帳號行銷策略與執行指引。本文件為策略層面；操作層面（訊息發送、回覆邏輯）見 company-ops 的 line-comms.md。

---

## 與 company-ops line-comms 的分工

| 面向 | line-comms（company-ops） | line-marketing（本文件） |
|------|--------------------------|------------------------|
| 定位 | 操作面 — 怎麼發、怎麼回 | 策略面 — 發什麼、發給誰、為什麼 |
| 涵蓋 | reply/reply_flex 工具使用、訊息格式、回覆流程 | 會員經營、分眾策略、行銷活動規劃 |
| 何時用 | 收到 LINE 訊息要回覆時 | 規劃 LINE 行銷活動、設計會員策略時 |

**系統可用的 LINE MCP 工具：**
- `reply(chat_id, text)` — 回覆文字訊息
- `reply_flex(chat_id, flex_json)` — 回覆 Flex Message（卡片、按鈕等）
- `mark_read(chat_id)` — 標記已讀
- `add_allowed_user(user_id, name)` — 新增允許的使用者
- `list_allowed_users()` — 列出所有允許的使用者


## LINE OA 基本功能

### 訊息類型

| 類型 | 說明 | 適用場景 | 系統工具 |
|------|------|---------|---------|
| 文字訊息 | 純文字，最基本 | 通知、回覆、簡短資訊 | `reply()` |
| Flex Message | 卡片式排版，可含按鈕、圖片 | 產品推薦、優惠券、問卷 | `reply_flex()` |
| 圖片訊息 | 單張圖片 | 新品照片、活動海報 | `reply()` + 圖片 URL |
| 影片訊息 | 短影片 | 產品示範、幕後花絮 | 需透過 LINE OA 後台 |
| 圖文訊息 | 圖片 + 文字 + 連結 | 文章推廣、活動導流 | 需透過 LINE OA 後台 |
| 多頁訊息 | 輪播卡片（最多 10 張） | 產品目錄、多選推薦 | `reply_flex()` 輪播格式 |

### 圖文選單（Rich Menu）

聊天室底部的常駐互動介面，是 LINE OA 最重要的入口設計。

**設計原則：**
- **6 格或 3 格** 為主流版型（6 格資訊量大，3 格按鈕大好點擊）
- **最重要的功能放左上角**（使用者視線起點）
- **明確的視覺層次**：用顏色區分不同功能區塊
- **文字精簡**：每格 2-4 個字為佳（如「最新優惠」「線上預約」「聯絡我們」）
- **定期更換**：配合檔期活動更新圖文選單

**常見格局配置（通用）：**

| 格位 | 功能建議 | 說明 |
|------|---------|------|
| 左上 | 最新優惠/主打活動 | 最醒目位置，放當期重點 |
| 右上 | 產品/菜單/服務介紹 | 核心資訊入口 |
| 左中 | 線上預約/點餐 | 轉換行為入口 |
| 右中 | 會員中心/集點 | 會員經營入口 |
| 左下 | 門市資訊/地圖 | 實體店導流 |
| 右下 | 聯絡客服 | 保留客服入口 |

### 自動回應

- **關鍵字回應**：設定常見問題的自動回覆（營業時間、地址、價目表）
- **AI 回應**：LINE OA 內建 AI 自動回應功能
- **歡迎訊息**：新好友加入時自動發送（建議含自我介紹 + 新好友優惠）
- **本系統角色**：透過 MCP 工具處理進階互動，自動回應處理基本問題


## 群發訊息策略

### 發送頻率建議

| 產業類型 | 建議頻率 | 說明 |
|---------|---------|------|
| 餐飲 | 每週 2-3 則 | 當日特餐、週末限定 |
| 零售 | 每週 1-2 則 | 新品上架、促銷活動 |
| 服務業 | 每週 1 則 | 預約提醒、保養通知 |
| B2B | 每月 2-4 則 | 產業資訊、產品更新 |

**核心原則：訊息品質 > 發送頻率**

### 最佳發送時段（台灣）

| 時段 | 適合類型 | 說明 |
|------|---------|------|
| 07:30-08:30 | 通勤族、上班族 | 通勤時段，開信率高 |
| 11:30-13:00 | 餐飲、午休消費 | 午餐決策時段 |
| 17:00-18:00 | 下班前消費決策 | 晚餐、下班採購 |
| 20:00-21:30 | 一般推播 | 晚間休閒，瀏覽意願高 |

### 分眾推播策略

**為什麼分眾 > 廣播：**
- 精準推播開封率可達 60-70%（廣播僅 20-30%）
- 減少封鎖率（收到不相關訊息 → 封鎖）
- 節省訊息費用（LINE OA 免費訊息有上限）

**常用分群維度：**

| 維度 | 標籤範例 | 應用 |
|------|---------|------|
| 消費頻率 | VIP、常客、新客、沉睡客 | 不同促銷力度 |
| 消費品類 | 咖啡愛好者、甜點控 | 精準推薦 |
| 地區 | 台北店、台中店 | 門市活動通知 |
| 互動行為 | 有點選優惠券、有回覆訊息 | 活躍度管理 |
| 加入管道 | 門市掃碼、線上活動 | 來源追蹤 |


## Flex Message 行銷應用

Flex Message 是 LINE 最強大的行銷訊息格式，可透過系統的 `reply_flex()` 工具發送。

### 常用模板

#### 產品卡片
```json
{
  "type": "bubble",
  "hero": {
    "type": "image",
    "url": "產品圖片URL",
    "size": "full",
    "aspectRatio": "20:13"
  },
  "body": {
    "type": "box",
    "layout": "vertical",
    "contents": [
      {"type": "text", "text": "產品名稱", "weight": "bold", "size": "xl"},
      {"type": "text", "text": "NT$XXX", "size": "lg", "color": "#FF5551"},
      {"type": "text", "text": "產品描述...", "size": "sm", "color": "#999999", "wrap": true}
    ]
  },
  "footer": {
    "type": "box",
    "layout": "vertical",
    "contents": [
      {"type": "button", "action": {"type": "uri", "label": "立即購買", "uri": "https://..."}, "style": "primary"}
    ]
  }
}
```

#### 優惠券
- 包含優惠內容、使用期限、使用條件
- 底部放「立即使用」按鈕
- 可加入條碼/QR Code 供門市核銷

#### 問卷/投票
- 用按鈕式選擇收集客戶偏好
- 結果自動記錄為標籤，用於後續分眾
- 簡短為佳（2-3 題），降低放棄率

#### 預約確認卡
- 顯示預約時間、服務項目、注意事項
- 提供「修改預約」「取消預約」按鈕
- 預約前 24 小時自動發送提醒

### Flex Message 設計原則

1. **一則訊息一個目標** — 不要在一張卡片放太多 CTA
2. **視覺層次清晰** — 標題 > 價格 > 描述 > 按鈕
3. **按鈕文字要行動導向** — 「立即購買」「馬上預約」而非「了解更多」
4. **行動裝置優先** — 所有設計在手機上測試
5. **載入速度** — 圖片壓縮，避免過大檔案


## LINE 會員經營

### 標籤分群系統

**自動標籤（系統行為觸發）：**
- 加好友來源（門市 QR/線上活動/廣告）
- 首次互動日期
- 最近互動日期
- 消費紀錄（已消費/未消費）

**手動標籤（互動收集）：**
- 問卷回覆結果
- 偏好選擇（透過圖文選單/Flex Message 按鈕）
- 客服互動紀錄
- VIP/常客標記

### 會員生命週期管理

| 階段 | 定義 | 策略 | 訊息類型 |
|------|------|------|---------|
| 新好友 | 加入 7 天內 | 歡迎禮、品牌介紹、首購優惠 | 歡迎訊息 + 優惠券 |
| 互動期 | 加入 7-30 天 | 內容培養、產品教育 | 圖文訊息、教學內容 |
| 首購轉換 | 加入 30 天內未消費 | 限時優惠推動首購 | Flex 優惠券 |
| 活躍客 | 定期消費 | 新品推薦、會員專屬 | 個人化推薦 |
| VIP | 高頻/高額消費 | 專屬服務、提前通知、生日禮 | 專屬 Flex 卡片 |
| 沉睡客 | 60 天以上無互動 | 喚醒優惠、問卷了解原因 | 回歸優惠券 |
| 流失預警 | 90 天以上無互動 | 最後嘗試或清理名單 | 簡短關懷訊息 |

### 消費行為追蹤

透過 business-db 系統整合：
- `log_interaction()` — 記錄 LINE 互動行為
- `find_customer()` / `update_customer()` — 客戶資料與標籤管理
- `list_orders()` — 消費紀錄查詢
- `record_transaction()` — 交易記錄

**整合要點：**
- LINE user_id 與客戶資料庫對應
- 消費後自動更新標籤（消費次數、累計金額）
- 生日、紀念日自動提醒發送祝福


## LINE 行銷活動範例

### 新好友歡迎流程
```
加好友 → 歡迎訊息（品牌介紹 + 首購 9 折券）
         → Day 3：熱門商品推薦
         → Day 7：未消費 → 限時加碼優惠
         → Day 7：已消費 → 感謝 + 回購折扣
```

### 節慶行銷活動
```
檔期前 3 週：預告訊息（吊胃口）
檔期前 1 週：早鳥優惠（VIP 先行）
檔期開始：全面開賣 + 限時加碼
檔期中：追加推播（針對有點開但未購買的受眾）
檔期結束：最後倒數 + 感謝回顧
```

### 定期內容經營
```
每週一：本週推薦（產品/服務）
每週四：實用小知識（建立專業形象）
每月一次：會員專屬優惠
季度一次：滿意度調查
```


## 成效指標

### LINE OA 關鍵指標

| 指標 | 計算方式 | 健康基準 |
|------|---------|---------|
| 好友數成長率 | 新增好友 / 總好友 | 每月 > 3% |
| 封鎖率 | 封鎖數 / 總好友 | 單次推播 < 0.3% |
| 開封率 | 開啟數 / 發送數 | 分眾 60-70%，廣播 20-30% |
| 點擊率 | 點擊數 / 開啟數 | > 10% |
| 轉換率 | 轉換數 / 點擊數 | 視產業，通常 5-15% |
| 訊息成本效益 | 營收 / 訊息費用 | ROAS > 5x |

### 降低封鎖率的方法

1. **分眾推播** — 只發相關內容給相關受眾
2. **控制頻率** — 不要過度推播（每週上限依產業調整）
3. **提供價值** — 不是每則都賣東西，穿插實用內容
4. **時段選擇** — 避免深夜或清晨發送
5. **給退訂選項** — 讓想降低頻率的人選擇，好過被封鎖
