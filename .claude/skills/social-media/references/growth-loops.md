## referral-program


# Referral Program（推薦計畫）

本模組以成長工程師的角度運作，為零售、服務業、電商、軟體等跨產業設計過推薦與聯盟計畫。知道會複利的計畫和蒙塵的計畫差在哪。目標是建立真的會運轉的推薦系統——有正確的機制、觸發、誘因與衡量，讓客戶幫您做獲客。

## 開工前

**先檢查既有的品牌／行銷脈絡：**
問問題之前，先用 `query_knowledge(category='brand')` 和 `query_knowledge(category='marketing')` 查詢既有脈絡。

收集以下脈絡（沒提供就問）：

### 1. 產品與客戶
- 賣什麼？（產品、服務、訂閱、電商）
- 理想客戶是誰？他們愛您產品的什麼？
- 平均 LTV（客戶終身價值）？（決定誘因天花板）
- 其他通路的 CAC（客戶獲取成本）是多少？

### 2. 計畫目標
- 要的結果？（更多註冊、更多營收、品牌觸及）
- 是 B2C 還是 B2B？（機制不同）
- 要客戶推薦客戶，還是合作夥伴推廣您的產品？

### 3. 現況（若要優化）
- 目前的計畫是什麼？
- 關鍵指標？（推薦率、轉換率、活躍推薦者比例）
- 獎勵結構？
- 迴圈在哪裡斷？


## 本模組的運作模式

### Mode 1：設計新計畫
從零開始。建構完整推薦計畫——迴圈、誘因、觸發、衡量。

**流程：**
1. 定義推薦迴圈（4 階段）
2. 選擇計畫類型（客戶推薦 vs. 聯盟）
3. 設計誘因結構（什麼、何時、給誰）
4. 找出觸發時機（何時請對方推薦）
5. 規劃分享機制（推薦實際怎麼發生）
6. 定義衡量框架

### Mode 2：優化既有計畫
已有運作中的計畫但表現不佳。診斷迴圈斷在哪。

**流程：**
1. 依基準稽核目前指標
2. 找出具體弱點（認知低、分享率低、轉換低、獎勵摩擦）
3. 執行聚焦修正——不要一次全重設計
4. 衡量影響後再動下一個槓桿

### Mode 3：啟動聯盟計畫
和客戶推薦不同。聯盟夥伴是外部推廣者——部落客、KOL、互補商家、產業電子報——由佣金驅動，不是忠誠度。

**流程：**
1. 定義聯盟階層與佣金結構
2. 找出並招募首批聯盟夥伴
3. 建立聯盟 Toolkit（連結、素材、文案）
4. 設定追蹤與支付機制
5. Onboard 並啟動前 10 位聯盟夥伴


## 推薦 vs. 聯盟——選對機制

|  | 客戶推薦 | 聯盟計畫 |
|---|---------|---------|
| **誰推廣** | 您的既有客戶 | 外部夥伴、發行商、KOL |
| **動機** | 忠誠度、獎勵、社群貨幣 | 佣金、受眾契合 |
| **最適合** | B2C、零售、在地服務 | B2B、高 LTV 產品、內容密集利基 |
| **啟動** | 由 aha moment、里程碑觸發 | 主動招募、onboard |
| **支付時機** | 帳戶點數、折扣、現金 | 營收分成或每次轉換固定費用 |
| **CAC 影響** | 低——獎勵 < CAC | 變動——佣金比例決定 |
| **規模** | 隨使用者基數擴展 | 隨夥伴招募擴展 |

**經驗法則：**客戶熱情且社交性強，從客戶推薦開始；客戶是企業為團隊採購，從聯盟開始。


## 推薦迴圈

每個推薦計畫都跑同樣的 4 階段迴圈。任一階段弱，迴圈就斷。

```
[觸發時機 Trigger Moment] → [分享行為 Share Action] → [被推薦者轉換 Referred User Converts] → [獎勵發放 Reward Delivered] → [迴圈 Loop]
```

### Stage 1：觸發時機
這是您請客戶推薦的時機。時機就是一切。

**高訊號觸發時機：**
- **aha moment 之後** — 客戶首次體驗核心價值時（不是註冊時——太早）
- **里程碑之後** — 「您剛省下第 100 小時」／「您的第 10 個團隊成員加入了」
- **好客服之後** — 問題解決後的 NPS 提示 → 9-10 分就請推薦
- **續訂之後** — 願意續訂的客戶等於告訴您他們滿意
- **公開好評之後** — 客戶發 Twitter 稱讚您 → 跟進推薦連結

**無效做法：**Day 1 就問、onboarding 信裡問、每封信底部都問。

### Stage 2：分享行為
拿掉每一個可能的摩擦點。

- 預填分享訊息（可編輯，不鎖定）
- 個人推薦連結（不是通用優惠碼）
- 分享選項：Email 邀請、複製連結、社群分享、B2B 的 Slack/Teams 分享
- 消費型產品做手機優化
- 一鍵送出——不要手動複製貼上

### Stage 3：被推薦者轉換
被推薦者進到您的產品。然後呢？

- 個人化到達頁（"Your friend Alex invited you — here's your bonus..."）
- 到達頁上明顯可見獎勵
- 從到達頁到轉換全程追蹤推薦歸因
- 明確 CTA——不要讓對方到處找該做什麼

### Stage 4：獎勵發放
獎勵必須快且清楚。延遲的獎勵會打斷迴圈。

- 被推薦者一註冊就確認獎勵資格（不是等到付款）
- 立即通知推薦者——不要等月結
- 儀表板上看得到狀態（"2 friends joined — you've earned $40"）


## 誘因設計

### 單邊 vs. 雙邊

**單邊**（只獎勵推薦者）：當產品本身有強病毒鉤子、客戶已經夠熱情時用。每次推薦成本低。

**雙邊**（推薦者和被推薦者都獎勵）：需要同時克服兩邊慣性時用。成本較高、轉換較高。Dropbox 用這個成名。

**規則：**推薦率 <1% 用雙邊；>5% 單邊較賺。

### 獎勵類型

| 類型 | 最適用途 | 範例 |
|------|---------|------|
| 帳戶點數 | 訂閱／重複消費 | "Get $20 credit" |
| 折扣 | 電商／用量型 | "Get 1 month free" |
| 現金 | 高 LTV、B2C | "$50 per referral" |
| 功能解鎖 | Freemium（免費 + 進階方案） | "Unlock advanced analytics" |
| 身分／肯定 | 社群／忠誠度 | "Ambassador status, exclusive badge" |
| 慈善捐款 | Enterprise／使命導向 | "$25 to a cause you choose" |

**規模規則：**帳戶點數應 ≥ 首月價值的 10%。現金獎勵上限為首筆付款的 30%。依 LTV 和 CAC 建模獎勵規模，確保單位經濟正向。

### 階梯式獎勵（遊戲化）
當您想讓推薦者從 1 個推到 10 個：

```
1 referral  → $20 credit（$20 帳戶點數）
3 referrals → $75 credit (25/referral) + bonus feature（$75 點數，每次 $25，加解鎖加值功能）
10 referrals → $300 cash + ambassador status（$300 現金 + 品牌大使身分）
```

階梯要簡單。最多三級。每一級要有明顯進步感，不只是稍微好。


## 優化槓桿

不要亂優化。先診斷，再拉對的槓桿。

| 指標 | 基準 | 低於基準時 |
|------|------|-----------|
| 推薦計畫認知度 | 活躍用戶 >40% 知道存在 | 在產品內推廣、活化後 Email |
| 活躍推薦者（%） | 活躍用戶基礎的 5-15% | 改善觸發時機與可見性 |
| 推薦分享率 | 看到的人 20-40% 會分享 | 簡化分享流程、改善訊息 |
| 被推薦者轉換率 | 15-25%（vs. 有機 5-10%） | 改善到達頁、加入誘因 |
| 獎勵兌換率 | 30 天內 >70% | 降低摩擦、發提醒 |

### 提升推薦率
- 把觸發時機往前移（aha 之後，不是 90 天之後）
- 在成功狀態加推薦提示（「您剛達到 1,000 個聯絡人——要分享給同事嗎？」）
- 推薦計畫要在產品儀表板現身，不只在 Email
- 測試雙邊 vs. 單邊獎勵

### 提升被推薦者轉換
- 個人化到達頁（"Invited by [Name]"）
- 首屏呈現被推薦者的具體利益
- 降低註冊摩擦——他們是被推薦來的，是溫的，別讓他們繞圈
- 像付費流量到達頁一樣做 A/B 測試


## 關鍵指標

每週追蹤：

| 指標 | 公式 | 為什麼重要 |
|------|------|-----------|
| 推薦率 | 發出的推薦 / 活躍用戶 | 計畫健康度 |
| 活躍推薦者 % | 發出 ≥1 次推薦的用戶 / 總活躍用戶 | 參與深度 |
| 推薦轉換率 | 轉換的推薦 / 發出的推薦 | 被推薦流量的品質 |
| 推薦 CAC | 獎勵成本 / 推薦來的新客戶 | 計畫經濟 vs. 其他通路 |
| 推薦營收貢獻 | 被推薦客戶營收 / 總營收 | 對業務的影響 |
| 病毒係數 Viral Coefficient（K，K-factor 每用戶帶來的新用戶數） | 每位用戶推薦數 × 轉換率 | K >1 = 病毒成長 |


## 聯盟計畫啟動清單

若要啟動聯盟計畫：

**啟動前**
- [ ] 佣金結構定義好（營收 % 或每次轉換固定費用）
- [ ] Cookie 視窗設定（至少 30 天，B2B 建議 90 天）
- [ ] 聯盟追蹤平台選定（Impact、ShareASale、Rewardful、PartnerStack 或自建）
- [ ] 聯盟合約起草（建議法律審閱）
- [ ] 支付條件明確（門檻、頻率、方式）

**夥伴 Toolkit**
- [ ] 每位聯盟夥伴的唯一追蹤連結
- [ ] 預寫好的文案與 Email swipes
- [ ] 核可的圖片和 Banner 廣告
- [ ] 產品說明文件（如何向他們的受眾解釋）
- [ ] 針對聯盟流量優化的到達頁

**招募**
- [ ] 50 個目標聯盟夥伴名單（互補商家、電子報、部落客、代理商）
- [ ] 個人化觸及——不是罐頭的「加入我們的聯盟計畫」
- [ ] 10 位聯盟先跑試點再擴大


## Proactive Triggers（主動觸發）

不等使用者問就主動提出：

- **註冊時就問** → 立刻標記。新用戶還沒體驗價值就請他們推薦是轉換殺手。把觸發改到 aha moment 之後。
- **獎勵相對 LTV 太小** → 獎勵 <5% LTV 且推薦率低，數學就壞了。提出規模問題。
- **沒有獎勵通知系統** → 被推薦者轉換了但推薦者沒有立即通知，迴圈就斷。標記需要即時通知。
- **通用分享訊息** → 讀起來像行銷文案的預填訊息會被刪。標記並以第一人稱客戶語氣重寫。
- **到達頁之後沒有歸因** → 推薦追蹤停在首訪，但轉換需要多次 Session，推薦被低估了。標記追蹤缺口。
- **聯盟計畫沒夥伴包** → 聯盟夥伴沒有核可的文案和素材，會推得不準或根本不推。啟動前標記。

## free-tool-strategy


# Free Tool Strategy（免費工具策略）

本模組以成長工程師的角度運作，建構並推出免費工具，在沒有任何付費廣告下帶來數十萬訪客、數千線索、數百個反向連結。知道哪些點子有力、哪些浪費工程時間。目標是協助決定要做什麼、如何設計以最大化價值與線索捕捉、如何推廣讓人找得到。

## 開工前

**先檢查既有的品牌／行銷脈絡：**
問問題之前，先用 `query_knowledge(category='brand')` 和 `query_knowledge(category='marketing')` 查詢既有脈絡。

收集以下脈絡（沒提供就問）：

### 1. 產品與受眾
- 您的核心產品與買家？
- 理想客戶有什麼問題，是一個免費工具能鄰近解決的？
- 您的受眾搜尋什麼是您產品以外的？

### 2. 資源
- 可以投入多少工程時間？（小時、天、週）
- 有設計資源嗎，還是 no-code/模板？
- 上線後誰維護？

### 3. 目標
- 主要目標：SEO 流量、Lead Gen、反向連結，還是品牌認知？
- 贏看起來像什麼？（每月 X 線索、Y 反向連結、Z 有機訪客）


## 本模組的運作模式

### Mode 1：評估工具點子
有一個或多個點子，不確定該做哪個——或到底該不該做。

**流程：**
1. 用 6 因素評估框架幫每個點子打分
2. 依您的目標與資源找出最有潛力的點子
3. 投工程時間前先用關鍵字資料驗證

### Mode 2：設計工具
已決定要做什麼。設計以最大化價值、線索捕捉與分享度。

**流程：**
1. 定義核心價值交換（用戶輸入什麼 → 得到什麼）
2. 設計最小摩擦的 UX
3. 規劃線索捕捉：在哪收、問什麼、漸進式 Profiling
4. 設計可分享的輸出（結果頁、生成報告、可嵌入徽章）
5. 規劃 SEO 到達頁結構

### Mode 3：上線與衡量
已建好。發佈並追蹤是否奏效。

**流程：**
1. 上線前：SEO 到達頁、Schema markup、提交到目錄
2. 上線通路：Product Hunt、Hacker News、產業電子報、社群
3. 外展：誰連結到類似工具？→ 建立連結取得名單
4. 衡量：設定使用量、線索、有機流量、反向連結的追蹤
5. 迭代：使用資料告訴您該改什麼


## 工具類型與何時用

| 工具類型 | 做什麼 | 開發複雜度 | 最適合 |
|---------|-------|-----------|-------|
| **Calculator（計算器）** | 輸入得出數字或區間 | 低—中 | LTV、ROI、定價、薪資、儲蓄 |
| **Generator（生成器）** | 生成文字、點子或結構化內容 | 低（模板）—高（AI） | 主標、Bio、文案、命名、報告 |
| **Checker（檢查器）** | 分析 URL、文字或檔案並評分／稽核 | 中—高 | SEO 稽核、可讀性、合規、拼字 |
| **Grader（評分器）** | 依量表評分 | 中 | 網站評分、Email 評分、銷售頁評分 |
| **Converter（轉換器）** | 把輸入從一種格式轉成另一種 | 低—中 | 單位、格式、貨幣、時區 |
| **Template（模板）** | 預建可填寫的文件 | 極低 | 合約、Brief、簡報、Roadmap |
| **Interactive Visualization（互動視覺化）** | 視覺化展示資料或概念 | 高 | 市場地圖、比較圖、趨勢資料 |


## 6 因素評估框架

每個點子在每個因素打 1-5 分。總分最高 = 先做。

| 因素 | 檢查什麼 | 1（弱） | 5（強） |
|------|---------|--------|--------|
| **搜尋量** | "free [tool]" 的每月搜尋量 | <100/月 | >5k/月 |
| **競爭** | 現有免費工具的品質 | 有優秀工具 | 沒有好的免費替代方案 |
| **開發成本** | 工程時間需求 | 數月 | 數天 |
| **線索捕捉潛力** | 能否自然地設 Gate 或收 Email？ | 強迫 Gate 殺 UX | 自然契合（結果寄 Email、報告下載） |
| **SEO 價值** | 能否建主題權威 + 反向連結？ | 薄、單頁工具 | 深度使用情境、連結磁鐵 |
| **病毒潛力** | 用戶會分享結果或嵌入工具嗎？ | 沒人分享 | 結果天生可分享 |

**評分指南：**
- 25-30：立即做
- 18-24：強候選，先驗證關鍵字量
- 12-17：資源低或契合策略缺口才做
- <12：放棄，或重新想概念


## 設計原則

### 先給價值再設 Gate
先給核心價值。Gate 放在升級上——更深報告、儲存結果、Email 寄送。如果只有給 Email 才有價值，那您做的是 Lead 表單，不是工具。

**好：**立刻顯示分數 → 提供寄完整報告
**壞：**"Enter your email to see your results"

### 最小摩擦
- 最多 3 個輸入得出初步結果
- 核心價值不需要註冊帳號
- 漸進式揭露：先簡單，詳細按需呈現
- 手機優化——50%+ 工具流量來自手機

### 可分享的結果
設計成讓用戶想分享：
- 獨特結果 URL，他人可造訪
- "Tweet your score"／"Copy your results" 按鈕
- 徽章或小工具的嵌入碼
- 可下載報告（PDF 或 CSV）
- 社群適用的圖片生成（分數卡、證書）

### 手機優先
- 輸入能在觸控螢幕運作
- 結果在手機上正確渲染
- 分享按鈕觸發原生分享面板
- 沒有依賴 Hover 的 UI


## 線索捕捉——何時、問什麼、怎麼問

### 何時設 Gate

**用 Email 設 Gate 當：**
- 結果複雜到能包裝成「報告」
- 工具產生持續價值（長時間追蹤、每月重跑）
- 結果個人化，用戶自然想保存

**不要設 Gate 當：**
- 核心結果是單一數字或短答案
- 競爭對手提供相同但無 Gate
- 主要目標是 SEO／反向連結（Gate 傷停留時間與連結）

### 問什麼

問最少的。每多一欄填寫率降 ~10%。

**第一層 Gate：**只問 Email
**第二層 Gate（重用或下載報告時）：**姓名 + 公司規模 + 角色

### 漸進式 Profiling
不要一次問完。跨多次 Session 建檔：
- Session 1：Email 存結果
- Session 2：角色、使用情境（情境式問、非表單）
- Session 3：公司、團隊規模（若他們要團隊功能）

## 衡量

從第一天就追蹤：

| 指標 | 告訴您什麼 | 工具 |
|------|-----------|------|

| 線索轉換率 | 有在產生線索嗎？ | CRM + GA4 events |
| 有機流量 | 有在排名嗎？ | Google Search Console |
| 推薦網域 | 有在賺連結嗎？ | Ahrefs / Google GSC |
| Email 轉付費轉換 | 有在產生 Pipeline 嗎？ | CRM 歸因 |
| 跳出率／停留時間 | 工具真的被用嗎？ | GA4 |

**上線 90 天目標：**
- 有機流量：每月 500+ Session
- 線索轉換：完成者 5-15%
- 推薦網域：10+ 有機反向連結

依您的流量與轉換假設建模損益兩平時間軸。


## Proactive Triggers（主動觸發）

不等使用者問就主動提出：

- **工具需要註冊帳號才能用** → 標記並重設計 Gate。這會殺 SEO、殺病毒、告訴用戶您在收割資料而非提供價值。
- **沒有可分享輸出** → 結果只存在 Session 裡、不能分享或儲存，就只做了半個工具。標記錯失的病毒機會。
- **沒有關鍵字驗證** → 工具概念在開發前沒對搜尋量驗證過就標記——3 小時研究勝過 3 週做沒人搜的工具。
- **競品已有相同的免費工具** → 既有工具穩固且免費時，門檻就是「10 倍更好否則別做」。標記競爭風險。
- **單一輸入 → 單一輸出** → 超簡工具很快失去 SEO 價值、不吸引連結。標記工具是否需要更多深度才值得連結。
- **沒有維護計畫** → 免費工具會在呼叫的 API 改變或邏輯過時時死掉。啟動前標記需要維護負責人。

### Do
- 在 aha moment 之後才請用戶推薦——用戶體驗到核心價值後推薦意願最高
- 獎勵設定在首月價值的 10% 以上——太低沒有動力
- 確保超過 40% 的活躍用戶知道推薦計畫存在——在產品內和 email 多處曝光
- 免費工具先給價值再收集資料——gate 放在進階報告或結果儲存

### Don't
- 不要在用戶註冊第一天就要求推薦——還沒體驗價值的推薦沒有說服力
- 不要用 "Just checking in" 的 follow-up（affiliate 情境）——提供具體價值或不要寄
- 不要用通用分享訊息——聽起來像行銷文案的推薦訊息會被忽略，用第一人稱客戶語言
- 不要單一通路依賴——口碑、付費、SEO、LINE 至少經營 2-3 個通路

## 快速參考

### 推薦計畫設計
1. 定義推薦迴圈四階段：觸發時機 → 分享行為 → 被推薦者轉換 → 獎勵發放
2. 選擇單邊或雙邊獎勵（推薦率 < 1% 用雙邊）
3. 設定觸發時機（aha moment 後、里程碑達成後、好評之後）
4. 追蹤關鍵指標：推薦率、轉換率、K 因子

### 免費工具評估與上線
1. 用 6 因素評估框架打分（搜尋量、競爭、工程量、lead 潛力、SEO 價值、傳播力）
2. 25 分以上立即開發；18-24 分先驗證關鍵字
3. 上線前：SEO 落地頁 + Schema 標記 + 提交目錄 + 外展連結名單

### 多通路獲客預算分配
1. Entry（NT$30,000/月）：Meta 為主、Google 品牌詞為輔
2. Growth（NT$100,000/月）：Meta + Google + 再行銷 + LINE 廣告
3. 所有通路設 UTM → 每週檢查 CAC 和 ROAS → 預算向贏家傾斜
