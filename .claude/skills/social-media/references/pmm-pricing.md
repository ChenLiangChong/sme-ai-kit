## 定價策略（沿用自 pricing-strategy skill）


# 定價策略

本模組專注於 SaaS 定價與營收模式設計，目標是讓定價能捕捉您交付的價值、維持健康的轉換率，並隨著客戶一起成長。

定價不是數學 — 是定位。正確的價格不是「覆蓋成本再加毛利」，而是坐落在「客戶的次佳替代方案要多少錢」與「客戶相信自己得到的價值」之間的那個數字。大部分 SaaS 產品定價偏低，本模組的任務就是清楚且有依據地修正這件事。

## 開始前

**先檢查現有的品牌 / 行銷脈絡：**
在提問之前，先用 `query_knowledge(category='brand')` 和 `query_knowledge(category='marketing')` 檢查既有脈絡。

蒐集以下脈絡：

### 1. 現況
- 目前有定價嗎？若有：哪些方案、什麼價位、計費模式是什麼？
- 試用/免費轉付費的轉換率是多少？（若已知）
- 每位客戶的平均營收是多少？
- 月流失率是多少？

### 2. 業務脈絡
- 產品類型：B2B 還是 B2C？自助式還是業務導入？
- 客戶分群：誰是最佳客戶、誰是偶爾使用的輕度使用者？
- 競品：客戶會拿您跟誰比較、那些方案多少錢？
- 成本結構：服務一位客戶每月成本是多少？

### 3. 目標
- 您是在設計、優化、還是規劃漲價？
- 有任何限制嗎？（例：舊客保留原價、合約限制、通路夥伴毛利）

## 本模組的運作模式

### 模式 1：從零設計定價
尚未有定價模型，或要完全重建。我們會逐步處理價值指標選擇、方案層級結構、價位研究與定價頁設計。

### 模式 2：優化現有定價
定價已存在但轉換率低、擴張停滯，或客戶覺得價格不合理。我們會稽核現況、對標業界、找出具體改善點。

### 模式 3：規劃漲價
價格必須調升 — 因為通膨、價值提升或市場重新定位。我們會設計能提升營收又不損傷客戶關係的策略。


## 定價的三條軸線

每個定價決策都跨越三條軸線，三者都要做對。

```
         ┌──────────────────────────┐
         │  方案組合 (Packaging)    │  每個方案裡裝什麼？
         │  （客戶能拿到的東西）    │
         └─────────────┬────────────┘
                       │
         ┌─────────────┴────────────┐
         │  價值指標 (Value Metric) │  按什麼收錢？
         │  （隨什麼擴展計費）      │
         └─────────────┬────────────┘
                       │
         ┌─────────────┴────────────┐
         │  價格點 (Price Point)    │  定價多少？
         │  （實際的數字）          │
         └──────────────────────────┘
```

大部分團隊會直接跳到價位。那是倒著做。先鎖定價值指標，再定方案組合，最後再測試數字。


## 價值指標（Value Metric）選擇

價值指標決定定價如何隨客戶價值擴展。選錯了，不是把錢留在桌上，就是造成阻力扼殺成長。

### SaaS 常見價值指標

| 指標 | 最適用於 | 範例 |
|--------|---------|---------|
| **按席次 / 使用者** | 協作工具、CRM | Salesforce、Notion、Linear |
| **按用量** | API 工具、基礎設施、AI | Stripe、Twilio、OpenAI |
| **按功能** | 平台型、加購模組 | Intercom、HubSpot |
| **單一定額** | 主打「無限」、SMB 工具 | Basecamp、Calendly Basic |
| **按成果** | 高價值、ROI 可量測 | 按佣金計費的工具 |
| **混合** | 上述組合 | 大部分成熟 SaaS |

### 如何選擇

回答以下問題：

1. **什麼會讓客戶願意付更多？** → 那就是您的價值指標
2. **這個指標會隨他們成功而擴展嗎？** → 客戶成長，您也成長
3. **是否容易理解？** → 複雜度會扼殺轉換
4. **是否難以鑽漏洞？** → 客戶不應能輕易繞過

**紅旗警訊：**
- 在「一個 Power User 做所有事」的工具裡用「按席次」→ 席次不隨價值擴展
- 「單一定額」但不同客戶取得 10 倍差異的價值 → 您在補貼重度使用者
- 「按 API 呼叫次數」但呼叫量每週差異巨大 → 帳單無法預測 = 流失


## Good-Better-Best（三階定價策略）三層方案結構

三層是標準。不是因為傳統 — 是因為它錨定了感知。

### 方案層級設計原則

**入門層（Good）：**
- 承接「定價再高就會流失」的客群
- 有限制 — 可以是功能、用量或支援
- 不是免費。免費是另一種策略（Freemium，免費 + 進階方案），不是方案層級。
- 至少應能覆蓋您的成本

**中間層（Better）— 您的主推方案：**
- 這是您要把大部分客戶導進來的層級
- 價格：入門層的 2-3 倍
- 功能：成長型公司需要的一切
- 視覺上明確標示為推薦方案

**頂層（Best）：**
- 給有企業需求的高價值客戶
- 可以是「洽詢價格」或客製定價
- 解鎖：SSO、Audit Log、SLA、專屬支援、客製合約
- 如果您有 MRR >$1k 的企業單，這一層就是為了承接它們

### 每層裝什麼

| 功能類別 | 入門方案 (Entry) | 進階方案 (Better) | 旗艦方案 (Best) |
|----------------|-------|--------|------|
| 核心產品 | ✅（有限） | ✅（完整） | ✅（完整） |
| 用量上限 | 低 | 中 | 高 / 無限 |
| 使用者 / 席次 | 1-3 | 5-無限 | 無限 |
| 整合 | 基礎 | 完整 | 完整 + 客製 |
| 報表 | 基礎 | 進階 | 客製 |
| 支援 | Email 支援 | 優先 | 專屬 CSM |
| 管理功能 | — | — | SSO、Audit Log、SCIM |
| SLA（Service Level Agreement 服務水準保證） | — | — | ✅ |


## 價值定價（Value-Based Pricing）

價格訂在「次佳替代方案」與「感知價值」之間。

```
[不做事的成本] ... [次佳替代方案] ... [您的價格] ... [感知交付價值]
```

**步驟 1：定義次佳替代方案**
- 如果您的產品不存在，客戶會怎麼做？
- 競品？Excel？手動流程？請人？
- 那要花客戶多少錢？

**步驟 2：估算交付價值**
- 省下的時間 × 使用者的時薪
- 產生或保住的營收
- 避免的錯誤成本 / 風險
- 問最佳客戶：「如果明天開始不能用我們的產品，您會損失什麼？」

**步驟 3：價格訂在中間**
- 粗略經驗法則：訂在可記錄交付價值的 10-20%
- 不要訂在價值的 50% — 客戶會覺得付太多
- 不要訂在次佳替代方案以下 — 等於告訴客戶您自己不相信產品

**用轉換率當訊號：**
- 試用到付費 >40%：很可能定價偏低 — 測試漲價
- 15-30%：大部分 SaaS 的健康水準
- <10%：定價可能偏高，或試用到付費的漏斗有阻力


## 定價研究方法

### Van Westendorp（價格敏感度測試）價格敏感度分析

對現有客戶或目標客群問這四題：

1. 什麼價格會讓您覺得這個產品便宜到懷疑品質？
2. 什麼價格會讓您覺得這個產品很划算 — 賺到了？
3. 什麼價格會讓您覺得開始變貴 — 但仍可接受？
4. 什麼價格會讓您覺得太貴不考慮？

**解讀結果：** 畫出四條曲線。「太便宜」與「太貴」的交點給出可接受價格區間。「划算」與「貴」的交點給出最佳價位。

**使用時機：** B2B SaaS，樣本數 n ≥ 30，現有客戶或合格潛在客戶。

### MaxDiff（最大差異量表，屬性偏好排序）分析

給受訪者看數組功能/價格，問他們最重視和最不重視哪個。統計分析會揭示各功能的相對價值 — 對方案組合的指引比對價位的指引更有用。

**使用時機：** 決定哪些功能放在哪個方案層時。

### 競品對標

| 步驟 | 做什麼 |
|------|-----------|
| 1 | 列出直接競品與客戶會考慮的替代方案 |
| 2 | 記錄他們公開的定價（方案名、價格、價值指標） |
| 3 | 記錄每個價位包含的內容 |
| 4 | 找出您在各項目上相對於競品的優勢與劣勢 |
| 5 | 依定位定價：溢價 = 高於市場 20-40%、價值 = 與市場持平或更低 |

**不要直接抄競品價格** — 他們的定價反映他們的成本結構與定位，不是您的。


## 漲價策略

漲價是 SaaS 公司可做的 ROI 最高的動作之一。大部分公司拖太久才動手。

### 策略選擇

| 策略 | 使用時機 | 風險 |
|---------|---------|------|
| **只漲新客** | 預期會遭遇強烈反彈 | 低 — 不動到既有客戶 |
| **既有客戶保留原價 + 延後漲** | 忠誠客戶基礎、合約風險 | 中 — 既有客戶感受到被尊重 |
| **搭配新價值交付** | 有明確新功能 / 改善 | 低 — 師出有名 |
| **方案重構** | 方案組合有重大變動 | 中 — 客戶需適應複雜度 |
| **全面調漲** | 對價值有信心、價格明顯低於市場 | 中至高 |

### 執行清單

1. **量化影響：** 用 100%、80%、70% 的既有客戶留存率，算出新 MRR
2. **依風險分群：** 年約、擁護者 vs. 批評者、用量型風險帳號
3. **訂日期：** 給既有客戶 60-90 天預告，最少 30 天
4. **說明理由：** 新功能、成本上升、投資於 [X] — 要具體
5. **提供出路：** 年約可鎖定原價，或給 3 個月緩衝窗口
6. **武裝 CS 團隊：** FAQ、話術、核准好的優惠權限
7. **監控 60 天：** 流失率、降級率、客服工單量

**20-30% 漲價的預期流失：** 5-15%。如果淨營收影響為正，就執行。


## 定價頁設計

定價頁是「意圖轉換為購買」的頁面。請以這個任務為核心來設計。

### 首屏（Above the Fold，首屏範圍，用戶無需滾動即可見）

必備：
- 方案名稱（簡單的：Starter / Pro / Enterprise，或以客戶分群命名）
- 價格與計費切換（月繳/年繳 — 年繳要顯示省多少）
- 每個方案 3-5 條差異化亮點
- 每個方案的 CTA 按鈕
- 推薦方案標註「Most popular」徽章

### 首屏以下

- **完整功能比較表** — 完整、易掃視，用 ✅ 和 ❌ 而非長段文字
- **FAQ 區** — 處理阻擋購買的 5 個異議：
  - 「可以隨時取消嗎？」
  - 「達到上限會怎樣？」
  - 「有退款嗎？」
  - 「我的資料安全嗎？」
  - 「如果要升級/降級怎麼辦？」
- **社群背書** — 每個方案對應的 Logo、客戶引言、案例
- **安全徽章** 若為 B2B 企業（SOC2、ISO 27001、GDPR）

### 年繳 vs. 月繳切換

- 預設顯示年繳價格（或視覺上突顯）— 提升 LTV
- 明確顯示省多少：「省 20%」或「2 個月免費」
- 不要隱藏月繳價格 — 隱藏會引發不信任


## Proactive Triggers（主動觸發）

主動提出以下警示，不必等使用者問：

- **試用到付費轉換率 >40%** → 強烈的定價偏低訊號。標記：測試漲價 20-30%。
- **所有客戶都在中間層** → 沒有向上升級路徑。標記：需要企業層，或功能鎖定不足。
- **客戶要求不在他們方案內的功能** → 擴張營收被放著不要。標記：檢視功能鎖定策略。
- **月流失率 >5%** → 先修流失，再漲價。漲價會加速本來就要流失的客戶離開。
- **價格兩年以上沒調** → 光通膨就足以支撐 10-15% 調漲。標記進策略檢視。
- **只有一個定價方案** → 沒有錨定、沒有升級。標記：加第三層，即使很少人買。

## 實體產品與服務業定價（台灣在地化補充）

> 以下內容適用於非軟體訂閱制的企業：實體產品製造、零售、批發、服務業。

### 成本加成定價法（Cost-Plus）

最基礎的定價方式，適用於製造業和零售業：

```
售價 = 原物料 + 包材 + 物流 + 倉儲 + 報廢耗損 + 目標毛利
```

| 產業 | 典型毛利率 | 備註 |
|------|-----------|------|
| 食品製造 | 30-50% | 含報廢率 5-10% |
| 服飾 | 50-70% | 含庫存壓力 |
| 專業服務 | 40-60% | 以人力成本為主 |
| B2B 零組件 | 15-30% | 薄利多銷 |

### 通路抽成結構（適用於實體產品進通路）

台灣實體通路的抽成結構會大幅影響定價：

| 通路 | 抽成範圍 | 額外費用 |
|------|---------|---------|
| 連鎖藥妝（屈臣氏/康是美） | 30-40% | 上架費、條碼費 |
| 便利超商（7-11/全家） | 35-45% | 檔期促銷贊助金 |
| 大型量販（好市多/家樂福） | 25-35% | 年度合約金 |
| 百貨專櫃 | 25-35% | 裝潢費、人事費 |
| 蝦皮/momo 電商 | 5-15% | 金流手續費、免運補貼 |

⚠️ 定價時必須從「終端售價」倒推，預留足夠的通路空間，否則毛利會被吃光。

### 心理定價（台灣消費者習慣）

- 尾數定價：NT$299 比 NT$300 感覺便宜很多
- 「均一價」：NT$39/49/99 店面在台灣很受歡迎
- 「滿額免運」：電商常用 NT$490/NT$990 門檻帶動客單價
- 「組合價」：買二送一、家庭包比單買便宜 15-20%

## Do's and Don'ts

### Do
- 定價是定位，不只是成本加利潤 — 先問「客戶認為值多少」再定價
- 使用 Good-Better-Best 三層架構，創造錨點和升級路徑
- 在上市前定好價格，價格錨點一旦設定就很難改
- 倒推通路抽成空間，確保每層都有合理毛利

### Don't
- 不要定價後才考慮競品 — 定價時就要知道替代方案的價格帶
- 不要所有客群用同一價格 — 不同客群的支付意願和價值感知不同
- 不要事後再調定價 — 價格錨點具有心理黏性，調漲比初始定價困難十倍
- 不要隱藏月費只推年費 — 隱藏會產生不信任感

## 快速參考

### 從零設計定價
1. 收集背景：產品類型、客群、競品價格、成本結構
2. 選定價值指標（Value Metric）— 客戶隨使用量增加而付更多的維度
3. 設計三層方案（Good / Better / Best），中間層為推薦方案
4. 用 Van Westendorp 或 MaxDiff 驗證價格敏感度
5. 設計定價頁面：推薦方案置中、年繳折扣明確標示

### 規劃漲價
1. 確認漲幅合理性（通膨、價值提升、市場重新定位）
2. 選擇漲價策略：新客先漲 / 到期漲 / 全面漲
3. 準備溝通範本：強調「新增價值」而非「成本上升」
4. 設定 90 天滾動執行計畫
5. 監控流失率，漲價後流失 > 5% 需立即檢視

### 實體產品定價（台灣）
1. 先確定目標終端售價（消費者願付價格）
2. 倒推通路抽成（藥妝 30-40%、超商 35-45%、電商 5-15%）
3. 扣除原物料 + 包材 + 物流 + 報廢後，確認毛利率是否足夠
4. 運用心理定價（尾數 99、滿額免運門檻、組合價）
