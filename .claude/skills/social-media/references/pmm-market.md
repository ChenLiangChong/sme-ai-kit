TAM / SAM / SOM、客群分類、趨勢分析、ICP 定義與國際市場進入。

## ICP 定義流程

定義理想客戶輪廓（ICP，Ideal Customer Profile）以鎖定目標：

1. 分析現有客戶（前 20% LTV）
2. 找出共同 Firmographics（規模、產業、營收）
3. 標記 Technographics（使用工具、成熟度、整合）
4. 記錄 Psychographics（痛感、動機、風險承受度）
5. 定義 3-5 個買家人物誌（經濟買家、技術買家、使用者）
6. 以銷售週期與流失資料驗證
7. 以 ICP 契合度為潛客評分 A/B/C/D
8. **驗證標準：** A 級契合客戶具備最低流失率與最快成交週期

### 台灣中小企業 Firmographics 範例（本土補充）

供服務傳產、批發、服務業、家族企業等客戶時參考：

| 維度 | 範圍範例 |
|------|---------|
| 規模 | 員工 5-50 人、年營收 1-5 億新台幣 |
| 成立階段 | 成熟期（10 年以上）、家族二代接班、傳產數位轉型中 |
| 區域 | 大台北、桃竹、中彰投、南高 / 或全台分銷 |
| 行業 | 製造業（塑膠/金屬/食品/紡織）、批發中盤、零售連鎖（< 20 間）、服務業（餐飲/美容/診所/補習）、B2B 貿易 |
| 資訊化程度 | 紙本 + Excel 混用、LINE 群組溝通、無 ERP 或僅有簡易 POS |
| 決策結構 | 老闆一人決策 / 二代與老闆共治 / 合夥制 |
| 預算 | 單次工具採購 1-20 萬、年 SaaS 訂閱 3-30 萬 |

**與 SaaS／美國 ICP 的差異：**
- 不看「融資輪次」，看「**年資 + 員工數 + 家族結構**」
- 通路不看 AWS/Snowflake 生態，看「**有無 ERP、會計軟體、POS**」
- 決策不看 CIO/VP Engineering，看「**老闆個人 + 會計 + 資深員工**」

### ICP 驗證檢核清單

- [ ] 5 位以上付費客戶符合此輪廓
- [ ] 銷售週期最短（低於中位數）
- [ ] LTV 最高（高於中位數）
- [ ] 流失率最低（年流失 < 5%）
- [ ] 產品互動強

## 市場規模框架

### TAM / SAM / SOM 計算

| 層級 | 定義 | 方法 |
|------|------|------|
| TAM（Total Addressable Market，總體可觸及市場） | 任何可能使用此類解決方案的人 | 產業報告 + Bottom-Up 估算 |
| SAM（Serviceable Addressable Market，可服務市場） | 以現有產品 / 通路可觸及的客群 | 由 ICP 條件過濾 TAM |
| SOM（Serviceable Obtainable Market，可獲取市場） | 12-24 個月內實際可拿下的份額 | SAM × 預期市場佔有率 % |

### Bottom-Up 估算模板

```
TAM = [潛在客戶數] × [平均年合約價值]
SAM = TAM × [符合 ICP 比例] × [可觸及地區比例]
SOM = SAM × [第一年合理市佔率（通常 1-5%）]
```

### 市場分類

| 客群 | 規模 | 成長 | 競爭強度 | 優先度 |
|------|------|------|---------|--------|
| [客群 1] | [# 客戶] | [% YoY] | [高/中/低] | [1-5] |
| [客群 2] | [# 客戶] | [% YoY] | [高/中/低] | [1-5] |
| [客群 3] | [# 客戶] | [% YoY] | [高/中/低] | [1-5] |

### 趨勢分析

1. 找出 3-5 個影響市場的宏觀趨勢
2. 將每個趨勢對應到機會或威脅
3. 量化影響時程（6 個月、1 年、3 年）
4. 對齊產品 Roadmap，乘順風車
5. 針對逆風建立防禦策略

## 市場研究方法

| 方法 | 速度 | 成本 | 深度 |
|------|------|------|------|
| 客戶訪談（5-10 位） | 1-2 週 | 低 | 高 |
| 問卷調查（100+ 回覆） | 2-3 週 | 低 - 中 | 中 |
| 產業報告（Gartner 等） | 立即 | 高 | 中 |
| 競品分析 | 1 週 | 低 | 中 |
| Social Listening | 持續 | 低 | 低 - 中 |
| 業務通話分析 | 1 週 | 低 | 高 |

### 客戶訪談指南

1. 「請說說您目前處理 [問題領域] 的工作流程」
2. 「最挫折的環節是什麼？」
3. 「您目前用什麼工具？缺了什麼？」
4. 「您如何衡量這個領域的成功？」
5. 「如果能揮舞魔法棒，您希望改變什麼？」
6. 「什麼會讓您從現行方案轉換？」
7. 「您今天在這件事上花多少時間 / 錢？」
8. 「還有誰參與這個決策？」

## Do's and Don'ts

### Do
- 用前 20% LTV 客戶來定義 ICP — 他們代表最高價值的目標客群
- ICP 驗證要有 5+ 付費客戶吻合，且具備最低流失率和最快成交週期
- 市場規模量化用 TAM/SAM/SOM 框架，附上資料來源與假設
- 結合多種研究方法（客戶訪談 + 產業報告 + 競品分析）交叉驗證

### Don't
- 不要假設「每個人都是我的客戶」— 沒有明確 ICP 會導致資源浪費
- 不要跳過 ICP 驗證直接上市 — 未驗證的 ICP 是假設，不是策略
- 不要用直覺代替資料驅動的客群分析 — 至少做 5-10 場客戶訪談
- 不要只看 TAM 就決定進入市場 — SOM 才是 12-24 個月內的實際機會

## 快速參考

### 定義 ICP
1. 分析現有客戶前 20% LTV，找出共同特徵（規模、產業、營收）
2. 標記技術成熟度（用什麼工具、整合需求）和心理特徵（痛感程度、風險偏好）
3. 建立 3-5 個買家人物誌（經濟買家、技術買家、使用者）
4. 用銷售週期和流失率資料驗證：A 級客戶應有最低流失 + 最快成交

### 計算市場規模
1. TAM = 潛在客戶數 x 平均年合約價值
2. SAM = TAM x ICP 符合比例 x 可觸及地區比例
3. SOM = SAM x 第一年合理市佔率（通常 1-5%）
4. 附上資料來源與假設，每季重新檢視

### 執行客戶訪談
1. 準備 8 題結構化問題（現況流程、最大痛點、使用工具、決策者）
2. 訪談 5-10 位目標客群，記錄逐字稿
3. 歸納共同模式，驗證 ICP 假設
4. 整理成 Market Analysis Document 存檔
