2事前準備

課前需要完成以下準備。建議講師協助學員逐一設定。

A. LINE Official Account + Messaging API

建立 LINE Business Account
到 manager.line.biz 建立帳號，選免費方案即可。
進入 LINE Developers Console
到 developers.line.biz/console/，建立 Provider。
建立 Messaging API Channel
在 Provider 下建立新的 Messaging API Channel。填入公司名稱、描述。
取得 Channel Access Token
Channel 設定頁 → Messaging API → Issue Channel Access Token（長效）。
複製保存，安裝時要用。
取得 Channel Secret
Channel 設定頁 → Basic settings → Channel Secret。
複製保存，安裝時要用。
開啟群組功能
Messaging API → Allow bot to join groups and multi-person chats → Enabled

B. ngrok 帳號

註冊 ngrok
到 ngrok.com 註冊免費帳號。
取得 Authtoken
Dashboard → Your Authtoken → 複製。
安裝時要用。
取得固定域名
Dashboard → Domains → 建立一個免費的固定域名（例：xxx-yyy.ngrok-free.dev）。
安裝時要用。

C. Claude 帳號

訂閱 Claude Max
到 claude.ai 註冊並訂閱 Max 方案（$100 USD/月）。
課前檢查清單 — 確認每位學員手上有以下四樣東西：

4首次訪談

安裝完成後，開啟 Claude。它會自動引導你完成公司設定。

# 啟動 Claude Code $ cd ~/sme-ai-kit && claude # 或開啟 Claude Desktop / Cowork，打開 sme-ai-kit 資料夾

Claude 會引導你完成

1. 公司基本資料

公司名稱、產業、營運時間、員工人數。存入資料庫，AI 會記住。

2. 員工名冊 + LINE 綁定

每位員工的姓名、職位、權限等級。掃 QR code 加 LINE OA 好友後綁定 LINE ID。

3. 商品 / 服務清單

SKU、名稱、庫存數量、安全庫存。Claude 會開始幫你追蹤。

4. 客戶 / 供應商 / 經銷商

名稱、電話、類型。可以之後慢慢補，不需一次到位。

5. 品牌語氣

對外回覆的語氣風格。給 Claude 看幾封範例信件或文案即可。

6. 營運規則 + 審核門檻

多少金額需要老闆核准？退貨規則？折扣政策？用講的就好。

反捏造機制：老闆講的規則會標記為「明確指示」並附上原話。Claude 自己推斷的會標記為「推斷」並主動確認。絕不會把推斷偽裝成老闆的指示。

7日常營運節奏

導入後的一天長什麼樣。

早上 — 開工
Claude 自動跑開機流程：營運摘要 + 庫存警報 + 逾期帳款。老闆打開 Desktop 看一眼就知道今天要做什麼。
白天 — LINE 處理
員工傳 LINE 回報進度、查庫存、提交報銷。客戶傳 LINE 詢問訂單。Claude 即時處理，需要老闆決定的自動轉發。
隨時 — 審核
超過門檻的操作（大額支出、行銷訊息、批量修改）→ Claude 建立審核請求 → LINE 通知老闆 → 老闆核准 → 自動執行。
傍晚 — 收工
Claude 自動保存每日快照（任務完成數、收支、庫存、客戶數）。隔天可以看趨勢。
月底 — 報表
「出月報」→ Claude 彙整所有數據，產出結構化報表。可匯出 PDF/Excel。

8注意事項

電腦不能休眠

daemon 跑在背景，電腦睡了就斷。Mac 建議設定「永不休眠」。

資料備份

data/business.db 是所有資料的核心。建議定期備份到雲端硬碟。

密鑰安全

.mcp.json 包含 LINE 密鑰，已加入 .gitignore。不要手動上傳到任何地方。

更新

講師推送更新後，學員只需要 git pull。DB 和設定不會被覆蓋。

LINE 免費方案限制：每月 200 則免費推播訊息。員工主動傳給 bot 的不算。如果需要大量群發，考慮升級 LINE OA 方案。

Q常見問題

LINE 訊息沒有收到？

檢查 daemon 是否在跑（ps aux | grep claude）。檢查 ngrok 是否正常（curl https://你的域名/webhook）。

Claude 回覆很慢？

正常。Claude 需要讀取技能、查資料庫、組織回覆。第一次回覆可能需要 5-10 秒。

可以多人同時用嗎？

可以。SQLite WAL mode 支援多人同時讀寫。Claude Code daemon 處理 LINE，Claude Desktop 老闆用，不衝突。

資料會上傳到 Anthropic 嗎？

對話內容會經過 Claude API（Anthropic 的伺服器），但資料庫存在你本地電腦。Anthropic 不會用 Claude Max 的對話訓練模型。

想加新功能怎麼辦？

跟 Claude 說就好。它可以讀 skills 裡的知識，也可以直接操作資料庫。複雜需求聯繫講師。
