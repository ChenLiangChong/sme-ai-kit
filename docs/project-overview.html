1. 產品定位

SME-AI-Kit 是給台灣中小企業的一套「AI 營運中樞」。
它不是 ERP 替代品，而是「從沒系統到有系統」的跳板—— 老闆用自然語言跟 Claude 對話，就能把客戶、訂單、庫存、帳務、任務、LINE 客服全部接起來。

51business-db tools
19social tools
7LINE tools
34Skill 模組

誰在用它

  • 老闆 — 用 Claude Desktop 的 Cowork 模式，隨時問「今天有什麼事」、「上個月毛利多少」
  • 員工 — 透過 LINE 直接跟 AI 互動：建任務、記支出、查庫存
  • 客戶 — 透過 LINE Official Account 下單、問價、查物流；陌生人由 AI 分流給對應窗口
  • 顧問 — 幫客戶導入時用 Claude Code CLI，透過訪談流程把公司 SOP 灌進 DB

7. 多事業體（business_unit）運作

一間公司同時經營多個品牌 / 部門（例如：內容產品、經銷物流、實體空間），系統用 business_unit 欄位把資料貫穿整個鏈路。

層級行為
LINE OA每個 OA 在 line-channels.json 綁定一個 business_unit；訊息 meta 自動帶入，Claude 不用問
訂單 / 帳務 / 任務 / 庫存建立時都要帶 business_unit；查詢時可按事業體篩選
客戶條件set_customer_entity_terms 設定「此客戶在事業體 X」的折扣率與付款條件，覆寫客戶預設
審核門檻business_entities.approval_threshold 優先於公司預設
規則全域規則（公司層級）+ 事業體專屬規則，query_knowledge 自動合併
員工employees.business_units（逗號分隔）限制能操作哪些事業體
舉例：同一個客戶「大創文具」對「WFA 事業體」有 8 折、30 天票期；對「經銷物流事業體」是 9.5 折、現金交易。 用 set_customer_entity_terms(customer_id=123, business_unit='brand_d', discount_rate=0.8, payment_terms='net30') 即可設定。

8. 使用情境（8 則真實流程）

情境 1　老闆早上打開 Claude Desktop

角色：老闆｜入口：Cowork｜頻率：每天早上

1
老闆：「早安，今天有什麼事？」
2
AI：自動載入 ops-dashboard 模組，並行跑 get_context_summary + low_stock_alerts + check_overdue
3
AI 回報：「進行中訂單 3 張、今天要收的逾期款 NT$45k、水貝土庫存剩 8 袋低於警戒、昨天有 2 則待審核」
4
老闆：「審核那兩則是什麼？」→ AI 展開 approvals 詳情
5
老闆：「第一筆核准」→ AI 呼叫 resolve_approval(id=12, 'approved')，然後執行原本被擋下的動作

情境 2　客戶透過 LINE 下單

角色：既有客戶｜入口：LINE OA（brand_d 事業體）｜處理者：CLI daemon

1
客戶傳「再來 20 袋水貝土，老地方送」
2
AI：訊息 meta 帶 channel_id=brand_d + business_unit=brand_d；find_customer 比對 LINE user_id → 認出是 #43 大創文具
3
AI：check_stock('水貝土', business_unit='brand_d') → 庫存 28 袋，OK
4
AI：讀取 customer_entity_terms → 折扣 8 折、net30；讀 #43 上次送貨地址
5
AI：create_order(customer_id=43, items=[...], business_unit='brand_d') → 回覆 Flex Message 確認明細 + 「打錯請回『取消』」
6
AI：create_task(title='出貨：大創文具 20 袋水貝土', assignee='倉庫組', business_unit='brand_d')

情境 3　員工 LINE 記一筆支出

角色：員工 小美｜入口：LINE（已綁定 employee）｜處理者：CLI daemon

1
小美傳「剛油錢 NT$1,850 統編發票」+ 一張發票照片
2
AI：認出是員工；照片下載到 media/line/images/
3
AI：用 Read tool 看發票 → 確認金額與日期；呼叫 record_transaction(type='expense', amount=1850, category='油費', attachments=[...], business_unit='brand_d', source_note='員工 LINE 回報')
4
AI：因小於審核門檻 5000，直接寫入；回覆「收到，已記入 WFA 事業體油費」

情境 4　陌生人 LINE 訊息的分層路由

角色：不認識的人｜入口：任一 OA

AI 不會直接回覆陌生人。流程由 line-comms 模組的陌生人路由定義：

1
判斷意圖：詢價／應徵／投訴／業務推銷／其他
2
按意圖查 business_rules 找對應負責人（例如：詢價 → 業務 阿哲；應徵 → HR 主任；投訴 → 老闆）
3
用 multicast 或內部 LINE 群組通知該負責人，附原訊息截圖
4
呼叫 mark_read 標記已處理；訊息存 line_messages 並打上待分派標籤
不能直接回覆陌生人的原因：AI 還不認識對方、不知道是不是競業、不知道老闆的商業敏感度。先讓人把關。

情境 5　HITL 審核（跨 session 核准）

11. 常見問答

這是 ERP 嗎？
不是。是「沒系統→有系統」的跳板。等業務規模撐起專業 ERP 時可以匯出遷移。
資料在哪？
單一 data/business.db（SQLite）+ data/media/ 資料夾。全部在老闆自己的機器，不上雲。
Claude 會亂編規則嗎？
不會。source_type='explicit' 強制帶原話；推斷的規則標 inferred 會顯示出來。
LINE 壞了怎麼辦？
Claude 啟動時會自動測 list_channels；失敗載入 setup 模組引導重設。
Context 爆掉怎麼辦？
壓縮前會寫 session_handoffs；下次啟動自動恢復。訂單等進行中工作都有 next_step_hint。
可以接 ERP 嗎？
可以。SQLite 可換 PostgreSQL，或為 business-db 換 adapter 打通外部 ERP API。
