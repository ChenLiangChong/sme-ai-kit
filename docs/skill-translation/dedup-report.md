## 方法論

對每檔做三步：

1. **掃出重疊**：檢查兩區塊中是否有 80% 字詞一致、或同一件事用兩種說法重複
2. **依職責歸位**：重複項若屬原則留 Do/Don't、屬步驟留快速參考
3. **保守原則**：不確定的留原位，只刪明顯重複

## 總體觀察

## 範例

### 一條刪除的重疊（email-outreach.md）

**刪除前**（快速參考 Email Sequence 設計流程）：
```
3. 規劃每封信：目的、寄送時間、主旨、預覽文字、內文、CTA
4. 每封信只做一件事（一個 CTA）、150 字以內、先講對方痛點  ← 刪除
5. 設定指標計畫：開信率、點擊率、轉換率基準
```

**刪除後**：
```
3. 規劃每封信：目的、寄送時間、主旨、預覽文字、內文、CTA
4. 設定指標計畫：開信率、點擊率、轉換率基準
```

Do 區塊仍保留：
- 「一封信只放一個 CTA——問越多、對方越不會做任何一件」
- 「控制在 150 字以內——手機螢幕兩屏是上限，超過就砍」

→ 原則只在 Do/Don't 出現一次，快速參考專注步驟。

### 一條保留的差異化（copy-editing.md）

**Do 區塊**：「七掃依序執行——每次 sweep 只聚焦一個維度（Clarity → Voice → So What → Prove It → Specificity → Emotion → Zero Risk）」

**快速參考「完整七掃編輯流程」**：展開每個 sweep 的具體判準（「每句話看得懂嗎？」「語氣一致嗎？」⋯⋯）

看似相同主題，但：
- Do/Don't 講「為什麼要依序、每次只聚焦一個」的**原則**
- 快速參考講「每個 sweep 實際要問什麼問題」的**操作**

兩邊資訊互補、不重疊，**保留**。
